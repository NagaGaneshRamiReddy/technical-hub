Rust Collections

Understanding Collections in Rust

Rust provides several powerful collection types for storing multiple values in a single data structure. The most common collections include Vectors, Strings, HashMaps, and HashSets.

1. Vectors

A Vector is a dynamically-sized array that allows you to store a sequence of elements. You can easily add, remove, and access elements:

fn main() {
    let mut vec = vec![1, 2, 3];
    vec.push(4);
    println!("{:?}", vec); // Outputs: [1, 2, 3, 4]
}

2. Strings

Rust's String type is a mutable sequence of characters. You can create a string and modify it:

fn main() {
    let mut my_string = String::from("Hello");
    my_string.push_str(", Rust!");
    println!("{}", my_string); // Outputs: Hello, Rust!
}

3. HashMaps

A HashMap is a collection of key-value pairs that allows for fast lookups. Hereâ€™s how to create and use one:

use std::collections::HashMap;

fn main() {
    let mut scores = HashMap::new();
    scores.insert(String::from("Team A"), 50);
    scores.insert(String::from("Team B"), 60);

    println!("{:?}", scores); // Outputs: {"Team A": 50, "Team B": 60}
}

4. HashSets

A HashSet is a collection of unique values. It is useful when you need to ensure that no duplicate values are stored:

use std::collections::HashSet;

fn main() {
    let mut unique_numbers = HashSet::new();
    unique_numbers.insert(1);
    unique_numbers.insert(2);
    unique_numbers.insert(1); // Duplicate value, will not be added

    println!("{:?}", unique_numbers); // Outputs: {1, 2}
}

Summary

Rust collections provide flexible and efficient ways to manage multiple values. Vectors, Strings, HashMaps, and HashSets each serve different purposes, allowing you to choose the right type for your data needs.
