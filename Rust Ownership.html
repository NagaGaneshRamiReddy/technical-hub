Rust Ownership

Understanding Ownership in Rust

Ownership is a fundamental concept in Rust that enables memory safety without the need for a garbage collector. It governs how memory is managed and ensures that resources are properly allocated and freed.

The Three Rules of Ownership

Rust’s ownership model is defined by three main rules:

  1. Each value in Rust has a variable that is its owner.
  2. A value can have only one owner at a time.
  3. When the owner of a value goes out of scope, Rust will automatically drop (deallocate) the value.

Example of Ownership

Here’s a simple example to illustrate ownership:

fn main() {
    let s1 = String::from("Hello"); // s1 owns the string
    let s2 = s1; // ownership of the string is transferred to s2

    // println!("{}", s1); // This will cause a compile-time error
    println!("{}", s2); // This works, as s2 now owns the string
}

In this example, when s1 is assigned to s2, ownership of the string is moved, and s1 can no longer be used.

Borrowing

To allow multiple variables to access the same data, Rust provides the concept of borrowing. You can borrow a value by using references:

fn main() {
    let s1 = String::from("Hello");
    let len = calculate_length(&s1); // borrowing s1

    println!("The length of '{}' is {}.", s1, len);
}

fn calculate_length(s: &String) -> usize {
    s.len() // s is a reference to a String
}

In this case, &s1 creates a reference to the string, allowing calculate_length to use it without taking ownership.

Mutable Borrowing

If you want to modify borrowed data, you need to ensure that there are no other active references. You can create a mutable reference:

fn main() {
    let mut s = String::from("Hello");
    change(&mut s); // mutable borrowing

    println!("Changed string: {}", s);
}

fn change(s: &mut String) {
    s.push_str(", world!");
}

Here, change takes a mutable reference to s, allowing it to modify the original string.
