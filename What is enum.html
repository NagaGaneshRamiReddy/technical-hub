What is an Enum in Rust?

Understanding Enums

In Rust, an enum (short for enumeration) is a powerful type that allows you to define a variable that can hold a value from a set of predefined options. Enums are particularly useful for representing a value that can be one of several distinct types.

Defining an Enum

To define an enum, use the enum keyword followed by the name of the enum and its variants:

enum Color {
    Red,
    Green,
    Blue,
}

In this example, we define a Color enum with three variants: Red, Green, and Blue.

Using Enums

Once defined, you can create instances of the enum variants:

fn main() {
    let my_color = Color::Red;

    match my_color {
        Color::Red => println!("The color is red."),
        Color::Green => println!("The color is green."),
        Color::Blue => println!("The color is blue."),
    }
}

Enums with Data

Enums can also store data associated with their variants. This allows you to define variants that hold different types of information:

enum Shape {
    Circle(f64), // Holds radius
    Rectangle(f64, f64), // Holds width and height
}

Here, the Shape enum has two variants: Circle, which holds a single f64 value (the radius), and Rectangle, which holds two f64 values (width and height).

Pattern Matching with Enums

Pattern matching is a key feature in Rust that allows you to handle different enum variants succinctly:

fn main() {
    let shape = Shape::Rectangle(5.0, 10.0);
    
    match shape {
        Shape::Circle(radius) => println!("Circle with radius: {}", radius),
        Shape::Rectangle(width, height) => println!("Rectangle {} x {}", width, height),
    }
}

Summary

Enums in Rust are a versatile and expressive way to create types that can have multiple variants, enhancing type safety and code clarity. They are often used in combination with pattern matching, allowing for flexible and readable code.
