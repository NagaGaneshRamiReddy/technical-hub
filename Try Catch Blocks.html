Try Catch Blocks in PHP

Overview

Try-catch blocks in PHP provide a mechanism to handle exceptions gracefully. This allows developers to manage errors and maintain the flow of the application.

Basic Structure

A try-catch block consists of a try block that contains code that may throw an exception, and a catch block that handles the exception:

    <?php
    try {
        // Code that may throw an exception
        throw new Exception("An error occurred!");
    } catch (Exception $e) {
        // Handle the exception
        echo "Caught exception: " . $e->getMessage();
    }
    ?>

Multiple Catch Blocks

You can use multiple catch blocks to handle different types of exceptions:

    <?php
    try {
        throw new InvalidArgumentException("Invalid argument!");
    } catch (InvalidArgumentException $e) {
        echo "Caught InvalidArgumentException: " . $e->getMessage();
    } catch (Exception $e) {
        echo "Caught exception: " . $e->getMessage();
    }
    ?>

Finally Block

A finally block can be added to execute code after the try-catch blocks, regardless of whether an exception was thrown:

    <?php
    try {
        throw new Exception("An error occurred!");
    } catch (Exception $e) {
        echo "Caught exception: " . $e->getMessage();
    } finally {
        echo "This will always run.";
    }
    ?>

Custom Exception Classes

You can create your own exception classes for more specific error handling:

    <?php
    class CustomException extends Exception {}

    try {
        throw new CustomException("This is a custom exception!");
    } catch (CustomException $e) {
        echo "Caught custom exception: " . $e->getMessage();
    }
    ?>

Error Reporting with Try-Catch

When using try-catch, you can also set error reporting to handle errors without disrupting the flow:

    <?php
    error_reporting(E_ALL);
    ini_set('display_errors', 1);

    try {
        echo $undefined_variable; // This will not stop the script
    } catch (Exception $e) {
        echo "Caught exception: " . $e->getMessage();
    }
    ?>
