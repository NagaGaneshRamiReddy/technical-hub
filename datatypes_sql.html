Primary and Foreign Keys

Primary and Foreign Keys

Primary Keys and Foreign Keys are fundamental concepts in relational database design that help ensure data integrity and establish relationships between tables.

Primary Keys

A Primary Key is a unique identifier for each record in a table. It ensures that each record can be uniquely identified and avoids duplication within the table. A primary key must contain unique values, and it cannot contain NULL values.

Primary Key Example Description 
Student
+-----------+-----------+
| StudentID | Name      |
+-----------+-----------+
| 1         | John Doe  |
| 2         | Jane Smith|
+-----------+-----------+
StudentID is the primary key for the Student table, uniquely identifying each student.

Foreign Keys

A Foreign Key is an attribute (or a set of attributes) in one table that refers to the primary key of another table. It establishes and enforces a link between the data in the two tables, ensuring referential integrity.

Foreign Key Example Description 
Enrollment
+-----------+-----------+-----------+
| StudentID | CourseID  | EnrollmentDate |
+-----------+-----------+-----------+
| 1         | 101       | 2024-01-15  |
| 2         | 102       | 2024-01-16  |
+-----------+-----------+-----------+
StudentID in the Enrollment table is a foreign key that refers to the StudentID primary key in the Student table.

Relationship between Primary and Foreign Keys

The primary key of one table is referenced by a foreign key in another table, creating a relationship between the two tables. This relationship helps maintain consistency and integrity across the database.

  • One-to-One (1:1) Relationship: A single record in one table is associated with a single record in another table. For example, a Person table and a Passport table where each person has one passport.
  • One-to-Many (1:N) Relationship: A single record in one table is associated with multiple records in another table. For example, a Department table and an Employee table where each department has many employees.
  • Many-to-Many (M:N) Relationship: Multiple records in one table are associated with multiple records in another table. This is typically implemented using a junction table. For example, a Student table and a Course table where students can enroll in multiple courses, and each course can have multiple students.

Design Considerations

  • Uniqueness: Ensure that primary keys are unique for each record to maintain data integrity.
  • Referential Integrity: Use foreign keys to enforce referential integrity and ensure that relationships between tables are valid.
  • Indexing: Consider indexing primary and foreign keys to improve query performance.
  • Consistency: Maintain consistency in primary key and foreign key naming conventions across the database schema.

Summary

Primary and Foreign Keys are essential elements in relational database design. A primary key uniquely identifies each record in a table, while a foreign key creates relationships between tables by referencing primary keys. Proper use of these keys helps maintain data integrity and enforce referential integrity in the database.
