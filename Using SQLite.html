Using SQLite in Flutter

Introduction to SQLite

SQLite is a lightweight, serverless, self-contained SQL database engine that is widely used in mobile applications. Flutter provides an easy way to integrate SQLite using the sqflite package.

1. Adding Dependencies

To get started with SQLite in Flutter, add the following dependencies to your pubspec.yaml file:

dependencies:
  sqflite: ^2.0.0+4
  path_provider: ^2.0.10

2. Setting Up the Database

First, create a helper class to manage database operations. Here’s how to set it up:

import 'package:sqflite/sqflite.dart';
import 'package:path/path.dart';

class DatabaseHelper {
  static final DatabaseHelper _instance = DatabaseHelper._internal();
  factory DatabaseHelper() => _instance;

  static Database? _database;

  DatabaseHelper._internal();

  Future get database async {
    _database ??= await _initDatabase();
    return _database!;
  }

  Future _initDatabase() async {
    String path = join(await getDatabasesPath(), 'my_database.db');
    return await openDatabase(
      path,
      onCreate: (db, version) {
        return db.execute(
          'CREATE TABLE users(id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT, age INTEGER)',
        );
      },
      version: 1,
    );
  }
}

3. Performing CRUD Operations

Now, let’s implement Create, Read, Update, and Delete operations:

Create a User

Future insertUser(User user) async {
  final db = await database;
  await db.insert(
    'users',
    user.toMap(),
    conflictAlgorithm: ConflictAlgorithm.replace,
  );
}

Read Users

Future> getUsers() async {
  final db = await database;
  final List> maps = await db.query('users');

  return List.generate(maps.length, (i) {
    return User.fromMap(maps[i]);
  });
}

Update a User

Future updateUser(User user) async {
  final db = await database;
  await db.update(
    'users',
    user.toMap(),
    where: 'id = ?',
    whereArgs: [user.id],
  );
}

Delete a User

Future deleteUser(int id) async {
  final db = await database;
  await db.delete(
    'users',
    where: 'id = ?',
    whereArgs: [id],
  );
}

4. User Model Class

Define a model class to represent user data:

class User {
  final int? id;
  final String name;
  final int age;

  User({this.id, required this.name, required this.age});

  Map toMap() {
    return {
      'id': id,
      'name': name,
      'age': age,
    };
  }

  factory User.fromMap(Map map) {
    return User(
      id: map['id'],
      name: map['name'],
      age: map['age'],
    );
  }
}

5. Complete Example: User Management

Here’s a complete example of a Flutter app that uses SQLite to manage user data:

import 'package:flutter/material.dart';

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: UserScreen(),
    );
  }
}

class UserScreen extends StatefulWidget {
  @override
  _UserScreenState createState() => _UserScreenState();
}

class _UserScreenState extends State {
  final DatabaseHelper _databaseHelper = DatabaseHelper();
  List _users = [];

  @override
  void initState() {
    super.initState();
    _loadUsers();
  }

  void _loadUsers() async {
    _users = await _databaseHelper.getUsers();
    setState(() {});
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('User Management')),
      body: ListView.builder(
        itemCount: _users.length,
        itemBuilder: (context, index) {
          return ListTile(
            title: Text(_users[index].name),
            subtitle: Text('Age: ${_users[index].age}'),
          );
        },
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () async {
          User newUser = User(name: 'Jane Doe', age: 30);
          await _databaseHelper.insertUser(newUser);
          _loadUsers();
        },
        child: Icon(Icons.add),
      ),
    );
  }
}

Expected Output:

The app will display a list of users stored in the SQLite database, with the ability to add new users.

Conclusion

Using SQLite in Flutter is an efficient way to manage local data. With the sqflite package, you can perform CRUD operations easily, making your app more dynamic and responsive to user interactions.
