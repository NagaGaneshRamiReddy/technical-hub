Transaction Control Commands

Transaction Control Commands in SQL

Transaction Control Commands are used to manage transactions within a database, ensuring that data modifications are performed accurately and consistently. These commands help control the transaction lifecycle and ensure data integrity by handling commit and rollback operations.

Key Transaction Control Commands

BEGIN TRANSACTION

The BEGIN TRANSACTION command starts a new transaction. All subsequent operations are part of this transaction until it is completed with a COMMIT or ROLLBACK.

BEGIN TRANSACTION;

COMMIT

The COMMIT command saves all changes made during the current transaction to the database. Once a transaction is committed, its changes become permanent and visible to other transactions.

COMMIT;

ROLLBACK

The ROLLBACK command undoes all changes made during the current transaction. It is used to revert the database to its previous state if an error occurs or if the transaction needs to be aborted.

ROLLBACK;

SAVEPOINT

The SAVEPOINT command sets a point within a transaction to which you can later roll back. This allows for partial rollback without affecting the entire transaction.

SAVEPOINT savepoint_name;

RELEASE SAVEPOINT

The RELEASE SAVEPOINT command removes a savepoint from the current transaction, allowing you to manage intermediate states effectively.

RELEASE SAVEPOINT savepoint_name;

SET TRANSACTION

The SET TRANSACTION command configures the properties of a transaction, such as its isolation level, which determines how transaction changes are visible to other transactions:

SET TRANSACTION ISOLATION LEVEL {READ UNCOMMITTED | READ COMMITTED | REPEATABLE READ | SERIALIZABLE};

Examples of Using Transaction Control Commands

Basic Transaction Example

Here is a simple example demonstrating the use of transaction control commands:

BEGIN TRANSACTION;

    -- Update account balances
    UPDATE Accounts
    SET Balance = Balance - 500
    WHERE AccountID = 'A';

    UPDATE Accounts
    SET Balance = Balance + 500
    WHERE AccountID = 'B';

    -- Commit the transaction if both updates succeed
    COMMIT;

Using Savepoints

Savepoints allow partial rollback within a transaction. For example, if an error occurs after a savepoint:

BEGIN TRANSACTION;

    -- Initial operations
    UPDATE Accounts
    SET Balance = Balance - 500
    WHERE AccountID = 'A';

    SAVEPOINT sp1;

    -- Intermediate operations
    UPDATE Accounts
    SET Balance = Balance + 500
    WHERE AccountID = 'B';

    -- If an error occurs, rollback to the savepoint
    IF @@ERROR != 0
    BEGIN
        ROLLBACK TRANSACTION TO SAVEPOINT sp1;
    END
    ELSE
    BEGIN
        COMMIT;
    END

Transaction Control in Different SQL Dialects

SQL Server

In SQL Server, use BEGIN TRANSACTION, COMMIT, ROLLBACK, SAVEPOINT, and SET TRANSACTION to manage transactions:

BEGIN TRANSACTION;
    -- Transaction operations
    COMMIT;

MySQL

In MySQL, transaction control is managed using similar commands:

START TRANSACTION;
    -- Transaction operations
    COMMIT;

Oracle

In Oracle, transactions are typically managed using COMMIT and ROLLBACK, with savepoints for intermediate rollback:

BEGIN;
    -- Transaction operations
    SAVEPOINT sp1;
    -- More operations
    COMMIT;

Summary

Transaction Control Commands are essential for managing transactions in SQL. By using BEGIN TRANSACTION, COMMIT, and ROLLBACK, along with savepoints and isolation levels, you can ensure accurate, reliable, and consistent database operations.
