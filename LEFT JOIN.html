LEFT JOIN

SQL LEFT JOIN

The LEFT JOIN (or LEFT OUTER JOIN) operation in SQL is used to combine rows from two tables based on a related column, returning all rows from the left table and the matching rows from the right table. If there is no match, NULL values are returned for columns from the right table.

Basic Syntax

    SELECT columns
    FROM table1
    LEFT JOIN table2
    ON table1.common_column = table2.common_column;
  • columns - The columns you want to retrieve from both tables.
  • table1 - The left table in the join operation.
  • table2 - The right table in the join operation.
  • common_column - The column used to match rows between the two tables.

Examples of LEFT JOIN

1. Basic LEFT JOIN

Retrieve a list of employees and their department names, including employees who do not belong to any department:

    SELECT employees.emp_name, departments.dept_name
    FROM employees
    LEFT JOIN departments
    ON employees.dept_id = departments.dept_id;

This query joins the employees table with the departments table. It returns all employees and their department names. Employees who are not assigned to any department will have NULL values in the dept_name column.

2. LEFT JOIN with Filtering

Retrieve employees and their departments where the department name is 'Sales', including employees who do not belong to the 'Sales' department:

    SELECT employees.emp_name, departments.dept_name
    FROM employees
    LEFT JOIN departments
    ON employees.dept_id = departments.dept_id
    WHERE departments.dept_name = 'Sales' OR departments.dept_name IS NULL;

This query includes all employees assigned to the 'Sales' department and also those who are not assigned to any department.

3. LEFT JOIN with Aggregation

Retrieve a list of departments and the number of employees in each department, including departments with no employees:

    SELECT departments.dept_name, COUNT(employees.emp_id) AS num_employees
    FROM departments
    LEFT JOIN employees
    ON departments.dept_id = employees.dept_id
    GROUP BY departments.dept_name;

This query joins the departments table with the employees table and counts the number of employees in each department. Departments with no employees will have a count of zero.

Advanced Usage of LEFT JOIN

1. LEFT JOIN with Multiple Tables

Retrieve a list of students, their courses, and the instructors for each course, including students who are not enrolled in any courses:

    SELECT students.student_name, courses.course_name, instructors.instructor_name
    FROM students
    LEFT JOIN enrollments ON students.student_id = enrollments.student_id
    LEFT JOIN courses ON enrollments.course_id = courses.course_id
    LEFT JOIN instructors ON courses.instructor_id = instructors.instructor_id;

This query joins the students, enrollments, courses, and instructors tables, showing students and their courses along with the instructors. Students not enrolled in any course will have NULL values in course_name and instructor_name.

2. LEFT JOIN with Sorting

Retrieve all employees and their department names, sorted by employee names:

    SELECT employees.emp_name, departments.dept_name
    FROM employees
    LEFT JOIN departments
    ON employees.dept_id = departments.dept_id
    ORDER BY employees.emp_name;

This query retrieves all employees and their departments, sorting the results by employee names.

Summary

The LEFT JOIN is a versatile SQL operation that retrieves all rows from the left table and the matching rows from the right table, with NULL values for non-matching rows. This join is useful for including records from one table regardless of matches in the other table.
