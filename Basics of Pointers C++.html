Basics of Pointer in C++

Introduction to Pointers

Pointers are variables that store the memory address of another variable. They are essential for tasks such as dynamic memory allocation, array manipulation, and implementing data structures. Understanding pointers is fundamental to mastering C++.

Declaring Pointers

To declare a pointer, you use the * symbol. The type of the pointer must match the type of the variable it points to.

#include <iostream>

int main() {
    int value = 10;    // Regular integer variable
    int *ptr;          // Pointer to an integer

    ptr = &value;      // Pointer stores the address of 'value'

    std::cout << "Value: " << value << std::endl;
    std::cout << "Address of value: " << &value << std::endl;
    std::cout << "Pointer: " << ptr << std::endl;
    std::cout << "Value through pointer: " << *ptr << std::endl;

    return 0;
}

In this example:

  • int value = 10; declares a regular integer variable.
  • int *ptr; declares a pointer to an integer.
  • ptr = &value; initializes the pointer with the address of the variable value.
  • *ptr dereferences the pointer to access the value at the address it points to.

Pointer Dereferencing

Dereferencing a pointer means accessing the value at the address stored in the pointer. This is done using the * operator.

    *ptr = 30; // Modify the value via the pointer

In this example:

  • *ptr = 30; modifies the value of num through the pointer ptr.

Null Pointers

A null pointer is a pointer that does not point to any valid memory location. It is often used to indicate that the pointer is not initialized or is intended to be unused.

#include <iostream>

int main() {
    int *ptr = nullptr; // Null pointer initialization

    if (ptr == nullptr) {
        std::cout << "Pointer is null." << std::endl;
    }

    return 0;
}

In this example:

  • nullptr is used to initialize a pointer to a null value.

Pointer Initialization

It is important to initialize pointers before using them to avoid undefined behavior. Uninitialized pointers can lead to unpredictable results.

#include <iostream>

int main() {
    int value = 50;
    int *ptr = &value; // Initialize pointer with the address of 'value'

    std::cout << "Pointer initialized to address: " << ptr << std::endl;

    return 0;
}

In this example:

  • ptr = &value; initializes the pointer with the address of value, ensuring it points to a valid memory location.
