Rust Traits

Understanding Traits in Rust

Traits in Rust are a way to define shared behavior in types. They allow you to specify functionality that can be implemented by different types, similar to interfaces in other programming languages. Traits enable polymorphism and code reuse.

Defining a Trait

You define a trait using the trait keyword. Here’s an example of a simple trait called Summary:

trait Summary {
    fn summarize(&self) -> String;
}

Implementing a Trait

After defining a trait, you can implement it for specific types. For example, here’s how to implement the Summary trait for a struct called Article:

struct Article {
    headline: String,
    content: String,
}

impl Summary for Article {
    fn summarize(&self) -> String {
        format!("{}: {}", self.headline, self.content)
    }
}

fn main() {
    let article = Article {
        headline: String::from("Rust Traits"),
        content: String::from("An introduction to traits in Rust."),
    };

    println!("Article Summary: {}", article.summarize());
}

Using Traits as Bounds

You can use traits as bounds on generic types, ensuring that the types used with a function implement the specified trait. Here’s an example:

fn notify(item: T) {
    println!("Breaking news! {}", item.summarize());
}

fn main() {
    let article = Article {
        headline: String::from("Using Traits in Rust"),
        content: String::from("Traits are powerful for code reuse."),
    };

    notify(article);
}

Default Implementations

Traits can provide default implementations for methods. Types that implement the trait can choose to use the default or override it:

trait Summary {
    fn summarize(&self) -> String {
        String::from("(Read more...)")
    }
}

struct Article {
    headline: String,
    content: String,
}

impl Summary for Article {
    fn summarize(&self) -> String {
        format!("{}: {}", self.headline, self.content)
    }
}

fn main() {
    let article = Article {
        headline: String::from("Default Implementation"),
        content: String::from("Traits can have default methods."),
    };

    println!("Article Summary: {}", article.summarize());
}

Traits for Shared Behavior

Traits can also be used to define shared behavior across different types. For instance, you can define a trait for drawing shapes:

trait Drawable {
    fn draw(&self);
}

struct Circle;
struct Square;

impl Drawable for Circle {
    fn draw(&self) {
        println!("Drawing a circle");
    }
}

impl Drawable for Square {
    fn draw(&self) {
        println!("Drawing a square");
    }
}

fn main() {
    let shapes: Vec<&dyn Drawable> = vec![&Circle, &Square];
    for shape in shapes {
        shape.draw();
    }
}

Summary

Traits are a fundamental feature in Rust that enable you to define shared functionality for different types. They enhance code reuse, promote polymorphism, and allow for better abstraction in your programs.
