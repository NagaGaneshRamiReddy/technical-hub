Rust Slices

Understanding Slices in Rust

Slices in Rust are a way to reference a contiguous sequence of elements in a collection, such as arrays or vectors, without taking ownership of the data. They provide a view into a portion of a collection and are useful for working with parts of data structures.

What is a Slice?

A slice is a reference to a contiguous sequence of elements. It is defined with a type syntax of &[T], where T is the type of elements in the slice. Slices can be created from arrays or vectors:

fn main() {
    let arr = [1, 2, 3, 4, 5];
    let slice = &arr[1..4]; // A slice of the array

    println!("Slice: {:?}", slice); // Outputs: [2, 3, 4]
}

Creating Slices

You can create a slice from any contiguous sequence of elements. Here are a couple of examples:

fn main() {
    let vec = vec![10, 20, 30, 40, 50];
    let slice_from_vec = &vec[0..3]; // Slice of the first three elements
    let slice_from_arr = &[100, 200, 300, 400][1..3]; // Slice from an array

    println!("Slice from vector: {:?}", slice_from_vec); // Outputs: [10, 20, 30]
    println!("Slice from array: {:?}", slice_from_arr); // Outputs: [200, 300]
}

Using Slices as Function Parameters

Slices are often used as parameters in functions to allow passing parts of collections without transferring ownership:

fn main() {
    let data = [1, 2, 3, 4, 5];
    print_slice(&data[1..4]); // Passing a slice

    let vec = vec![6, 7, 8, 9];
    print_slice(&vec[0..2]); // Passing another slice
}

fn print_slice(slice: &[i32]) {
    for &item in slice {
        println!("{}", item);
    }
}

Summary

Slices provide a way to work with sections of arrays and vectors in Rust without taking ownership of the data. They are lightweight and efficient, making them an essential tool for handling collections in a safe and performant manner.
