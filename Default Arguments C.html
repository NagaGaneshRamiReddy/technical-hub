Default Arguments in C++

Introduction

In C++, default arguments allow you to specify default values for function parameters. This means you can call a function without providing all of the arguments, and the function will use the default values for any missing arguments. Default arguments make functions more flexible and simplify code when certain parameters have common default values.

How Default Arguments Work

When you declare a function with default arguments, you provide default values for some or all of the parameters. If a caller omits arguments in a function call, the default values are used. Default arguments must be specified in the function declaration and can be used only from right to left.

Syntax

To specify default arguments, assign default values to the parameters in the function declaration:

return_type function_name(parameter1_type param1 = default_value1, parameter2_type param2 = default_value2, ...);

#include <iostream>

// Function declaration with default arguments
void greet(std::string name, std::string greeting = "Hello");

int main() {
    greet("Alice"); // Uses default greeting
    greet("Bob", "Hi"); // Uses provided greeting
    return 0;
}

// Function definition
void greet(std::string name, std::string greeting) {
    std::cout << greeting << ", " << name << "!" << std::endl;
}

In this example:

  • void greet(std::string name, std::string greeting = "Hello"); - The greet function has a default argument for the greeting parameter.
  • When greet("Alice") is called, the default value "Hello" is used for the greeting parameter.
  • When greet("Bob", "Hi") is called, the provided value "Hi" is used instead of the default.

Rules for Using Default Arguments

  • Rightmost Parameters: Default arguments can only be provided for parameters from the rightmost side. Once you provide a default value for a parameter, all parameters to its right must also have default values.
  • Declaration and Definition: Default values should be provided in the function declaration. They are not required in the function definition if the declaration already includes them.

Common Issues and Troubleshooting

If you encounter issues with default arguments:

  • Incorrect Default Argument Placement: Ensure default arguments are only used from the rightmost parameters. The compiler will raise errors if default arguments are not placed correctly.
  • Function Overloading Conflicts: Be cautious when overloading functions with default arguments, as it may lead to ambiguities in function calls. Make sure function calls are unambiguous.
