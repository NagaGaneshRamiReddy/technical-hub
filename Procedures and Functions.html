Procedures and Functions in PL/SQL

Introduction to Procedures and Functions

Procedures and Functions are two key components of PL/SQL that allow for modular programming. They enable developers to encapsulate logic and reuse code, promoting better organization and efficiency in database applications.

What is a Procedure?

A procedure is a stored PL/SQL program that performs a specific task. It can take parameters and can be invoked independently of other PL/SQL blocks.

Creating a Procedure


CREATE OR REPLACE PROCEDURE greet_employee (
    p_employee_name IN VARCHAR2
) AS
BEGIN
    DBMS_OUTPUT.PUT_LINE('Hello, ' || p_employee_name || '!');
END greet_employee;

Executing a Procedure


BEGIN
    greet_employee('John Doe');
END;

What is a Function?

A function is similar to a procedure, but it always returns a value. Functions are often used in SQL statements to compute values or perform calculations.

Creating a Function


CREATE OR REPLACE FUNCTION calculate_bonus (
    p_salary IN NUMBER
) RETURN NUMBER AS
BEGIN
    RETURN p_salary * 0.10; -- 10% bonus
END calculate_bonus;

Executing a Function


DECLARE
    v_bonus NUMBER;
BEGIN
    v_bonus := calculate_bonus(5000);
    DBMS_OUTPUT.PUT_LINE('Bonus: ' || v_bonus);
END;

Differences Between Procedures and Functions

  • Return Value: Functions return a single value; procedures do not.
  • Usage: Functions can be called within SQL statements; procedures cannot.
  • Purpose: Functions are primarily used for calculations, while procedures are used for performing actions.

Best Practices for Using Procedures and Functions

  • Keep procedures and functions focused on a single task to enhance clarity and maintainability.
  • Use meaningful names for procedures and functions that clearly describe their purpose.
  • Document your code with comments to explain complex logic.
  • Handle exceptions within procedures and functions to prevent unhandled errors during execution.

Conclusion

Understanding procedures and functions in PL/SQL is essential for effective database programming. They facilitate code reuse, improve organization, and enhance the performance of database applications.
