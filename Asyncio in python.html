Asyncio in Python

What is Asyncio ?

Asyncio is a library in Python used to write concurrent code using the async and await syntax. It provides support for asynchronous I/O, event loops, coroutines, tasks, and more. Asyncio is ideal for handling high-level structured network code and other I/O-bound tasks.

Key Concepts in Asyncio

  • Event Loop: The core of asyncio. It runs asynchronous tasks and callbacks, handles events, and manages the execution of coroutines.
  • Coroutines: Special functions defined with async def that can pause and resume their execution with await.
  • Tasks: Wrappers around coroutines to schedule their execution and track their progress.
  • Futures: Representations of results that will be available in the future. They can be awaited or used for coordination between coroutines.

Basic Usage

Hereâ€™s a basic example demonstrating the usage of asyncio:

# basic_asyncio.py
import asyncio

async def say_hello():
    print("Hello")
    await asyncio.sleep(1)
    print("World")

async def main():
    await say_hello()

if __name__ == "__main__":
    asyncio.run(main())

This code defines an asynchronous function say_hello that prints "Hello", waits for one second, and then prints "World". The main function awaits the completion of say_hello, and asyncio.run is used to run the main function in an event loop.

Examples

Running Multiple Coroutines

To run multiple coroutines concurrently, use asyncio.gather:

# multiple_coroutines.py
import asyncio

async def greet(name: str):
    print(f"Hello {name}")
    await asyncio.sleep(1)
    print(f"Goodbye {name}")

async def main():
    await asyncio.gather(
        greet("Alice"),
        greet("Bob"),
        greet("Charlie"),
    )

if __name__ == "__main__":
    asyncio.run(main())

Handling Timeouts

You can handle timeouts in asyncio using asyncio.wait_for:

# timeout_example.py
import asyncio

async def slow_operation():
    await asyncio.sleep(5)
    return "Finished"

async def main():
    try:
        result = await asyncio.wait_for(slow_operation(), timeout=2)
        print(result)
    except asyncio.TimeoutError:
        print("Operation timed out")

if __name__ == "__main__":
    asyncio.run(main())

Best Practices

  • Use Async Libraries: Prefer libraries that support async I/O for better performance with asynchronous code.
  • Avoid Blocking Code: Ensure that all code that could block the event loop is non-blocking. Use async I/O operations wherever possible.
  • Handle Exceptions: Properly handle exceptions in asynchronous code to prevent unexpected crashes.
  • Limit Concurrency: Be mindful of the number of concurrent tasks to avoid overwhelming resources.

Summary

Asyncio is a powerful library in Python that enables asynchronous programming, making it easier to handle I/O-bound and high-level structured network code. Understanding its core concepts and best practices will help you write efficient and scalable asynchronous applications.
