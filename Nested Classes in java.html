Nested classes in Java

Introduction

In Java, nested classes are classes defined within another class. They can be used to logically group classes and encapsulate functionality. Java supports several types of nested classes, each serving different purposes and having specific use cases.

1. Static Nested Classes

A static nested class is a static member of the outer class. It can only access the static members of the outer class and does not require an instance of the outer class to be created.

public class OuterClass {
    private static String staticField = "Static Field";

    static class StaticNestedClass {
        void display() {
            System.out.println("Static nested class can access: " + staticField);
        }
    }

    public static void main(String[] args) {
        StaticNestedClass nested = new StaticNestedClass();
        nested.display();
    }
}

2. Non-Static Nested Classes (Inner Classes)

Inner classes (non-static nested classes) are associated with an instance of the outer class. They can access both static and instance members of the outer class.

public class OuterClass {
    private String instanceField = "Instance Field";

    class InnerClass {
        void display() {
            System.out.println("Inner class can access: " + instanceField);
        }
    }

    public static void main(String[] args) {
        OuterClass outer = new OuterClass();
        InnerClass inner = outer.new InnerClass();
        inner.display();
    }
}

3. Local Classes

Local classes are defined within a method and are local to that method. They are useful for encapsulating small pieces of functionality that are only relevant to a specific method.

public class OuterClass {
    void method() {
        class LocalClass {
            void display() {
                System.out.println("This is a local class.");
            }
        }
        
        LocalClass local = new LocalClass();
        local.display();
    }

    public static void main(String[] args) {
        OuterClass outer = new OuterClass();
        outer.method();
    }
}

4. Anonymous Classes

Anonymous classes are used to create instances of classes that do not need to be reused. They are often used when you need to implement an interface or extend a class in a concise way.

public class OuterClass {
    void method() {
        Runnable runnable = new Runnable() {
            @Override
            public void run() {
                System.out.println("This is an anonymous class.");
            }
        };
        new Thread(runnable).start();
    }

    public static void main(String[] args) {
        OuterClass outer = new OuterClass();
        outer.method();
    }
}

Comparison of Nested Classes

  • Static Nested Classes: Do not require an instance of the outer class. Access only static members of the outer class.
  • Inner Classes: Require an instance of the outer class. Can access both static and instance members of the outer class.
  • Local Classes: Defined within a method and can only be used within that method.
  • Anonymous Classes: Provide a way to instantiate and use a class in one step, often used for quick implementations of interfaces or abstract classes.

Summary

Nested classes in Java provide a powerful way to group and organize classes within other classes. Understanding the different types of nested classes—static nested classes, inner classes, local classes, and anonymous classes—helps in choosing the appropriate type based on the requirements of encapsulation, organization, and usage scenarios.
