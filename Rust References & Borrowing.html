Rust References & Borrowing

Understanding References and Borrowing in Rust

References and borrowing are integral concepts in Rust that allow functions to access data without taking ownership, enabling memory safety and preventing data races.

What are References?

A reference is a pointer to a value, allowing you to refer to data without taking ownership. In Rust, references are denoted with an ampersand (&):

fn main() {
    let s = String::from("Hello");
    let len = calculate_length(&s); // Borrowing s

    println!("The length of '{}' is {}.", s, len);
}

fn calculate_length(s: &String) -> usize {
    s.len() // s is a reference to the String
}

In this example, &s borrows the string, allowing calculate_length to use it without taking ownership.

Immutable References

By default, references in Rust are immutable, meaning you cannot modify the borrowed data:

fn main() {
    let s = String::from("Hello");
    let r = &s; // Immutable reference

    // r.push_str(", world!"); // This would cause a compile-time error
    println!("{}", r);
}

Attempting to modify r results in a compile-time error because it's an immutable reference.

Mutable References

If you want to modify borrowed data, you can create a mutable reference:

fn main() {
    let mut s = String::from("Hello");
    change(&mut s); // Mutable borrowing

    println!("Changed string: {}", s);
}

fn change(s: &mut String) {
    s.push_str(", world!"); // Modifying the original string
}

Here, change takes a mutable reference, allowing it to modify the original string.

Rules of References

Rust enforces some important rules regarding references:

  1. You can have either one mutable reference or any number of immutable references at a time.
  2. References must always be valid. Rust ensures this at compile time.

Summary

References and borrowing in Rust allow for safe data access without ownership transfer. Understanding these concepts is crucial for effective memory management and writing safe Rust code.
