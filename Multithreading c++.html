Multithreading in C++

Introduction

Multithreading is a technique used in programming to run multiple threads concurrently within a single process. In C++, multithreading allows a program to perform several tasks at the same time, which can lead to better performance and responsiveness. It is particularly useful in applications that perform time-consuming operations, such as data processing or network communication.

Including the Thread Library

To use multithreading in C++, you need to include the thread library by adding the following line:

#include <thread>

Creating and Running Threads

You can create and run threads using the std::thread class. Here is a simple example:

#include <iostream>
#include <thread>

void printHello() {
    std::cout << "Hello from the thread!" << std::endl;
}

int main() {
    // Create a new thread that runs the printHello function
    std::thread t(printHello);
    
    // Wait for the thread to finish
    t.join();
    
    return 0;
}

In this example:

  • std::thread t(printHello); creates a new thread that runs the printHello function.
  • t.join(); waits for the thread to finish execution before the main thread continues.

Passing Arguments to Threads

Threads can also accept arguments. Here’s how you can pass arguments to a thread function:

#include <iostream>
#include <thread>

void printNumber(int num) {
    std::cout << "Number: " << num << std::endl;
}

int main() {
    int number = 42;
    // Create a new thread that runs the printNumber function with an argument
    std::thread t(printNumber, number);
    
    // Wait for the thread to finish
    t.join();
    
    return 0;
}

Synchronization

When multiple threads access shared resources, synchronization is necessary to avoid conflicts and ensure data consistency. The C++ Standard Library provides synchronization mechanisms such as mutexes:

#include <iostream>
#include <thread>
#include <mutex>

std::mutex mtx;

void printNumbers(int num) {
    std::lock_guard<std::mutex> lock(mtx);  // Lock the mutex for thread safety
    std::cout << "Number: " << num << std::endl;
}

int main() {
    std::thread t1(printNumbers, 1);
    std::thread t2(printNumbers, 2);
    
    t1.join();
    t2.join();
    
    return 0;
}

In this example:

  • std::mutex mtx; declares a mutex.
  • std::lock_guard<std::mutex> lock(mtx); automatically locks the mutex for the duration of the scope to ensure that only one thread accesses the critical section at a time.

Common Issues and Troubleshooting

When working with multithreading, be aware of common issues:

  • Race Conditions: Occur when two or more threads access shared resources concurrently without proper synchronization.
  • Deadlocks: Happen when two or more threads are waiting for each other to release resources, causing a standstill.
  • Thread Safety: Ensure that shared data is accessed in a thread-safe manner to prevent inconsistencies.
