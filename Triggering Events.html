Triggering Events

Triggering Events in SQL

In SQL, Triggering Events refer to the specific actions that cause a trigger to execute. Triggers are designed to respond to changes in a database table, such as insertions, updates, or deletions. Understanding these events is crucial for effective trigger implementation.

Types of Triggering Events

Triggers can be defined to activate in response to several types of events:

INSERT

An INSERT Trigger activates after a new record is added to a table. This is useful for actions such as auditing, validating, or automatically populating related tables:

CREATE TRIGGER trgAfterInsert
ON Employees
AFTER INSERT
AS
BEGIN
    INSERT INTO AuditLog (Action, EmployeeID, ActionDate)
    SELECT 'INSERT', EmployeeID, GETDATE()
    FROM inserted;
END;

UPDATE

An UPDATE Trigger executes when an existing record in a table is modified. It helps in tracking changes or enforcing constraints related to updates:

CREATE TRIGGER trgAfterUpdate
ON Employees
AFTER UPDATE
AS
BEGIN
    INSERT INTO AuditLog (Action, EmployeeID, ActionDate, OldSalary, NewSalary)
    SELECT 'UPDATE', i.EmployeeID, GETDATE(), d.Salary, i.Salary
    FROM inserted i
    JOIN deleted d ON i.EmployeeID = d.EmployeeID;
END;

DELETE

A DELETE Trigger is triggered when a record is removed from a table. It can be used to archive deleted data or prevent certain deletions:

CREATE TRIGGER trgAfterDelete
ON Employees
AFTER DELETE
AS
BEGIN
    INSERT INTO DeletedEmployees (EmployeeID, Name, TerminationDate)
    SELECT EmployeeID, Name, GETDATE()
    FROM deleted;
END;

Triggering Events in Different SQL Dialects

SQL Server

In SQL Server, triggers can be created for INSERT, UPDATE, and DELETE events using the AFTER, BEFORE, or INSTEAD OF keywords:

CREATE TRIGGER trgAfterInsert
ON Employees
AFTER INSERT
AS
BEGIN
    -- Trigger logic here
END;

MySQL

In MySQL, triggers can also be created for INSERT, UPDATE, and DELETE events using similar syntax:

CREATE TRIGGER trgBeforeUpdate
BEFORE UPDATE ON Employees
FOR EACH ROW
BEGIN
    -- Trigger logic here
END;

Oracle

In Oracle, triggers can respond to INSERT, UPDATE, and DELETE events, and can be defined as BEFORE, AFTER, or INSTEAD OF:

CREATE OR REPLACE TRIGGER trgBeforeInsert
BEFORE INSERT ON Employees
FOR EACH ROW
BEGIN
    -- Trigger logic here
END;

Examples of Triggering Events

Logging Inserts

Automatically log each new employee record to an audit table:

CREATE TRIGGER trgAfterInsert
ON Employees
AFTER INSERT
AS
BEGIN
    INSERT INTO AuditLog (Action, EmployeeID, ActionDate)
    SELECT 'INSERT', EmployeeID, GETDATE()
    FROM inserted;
END;

Tracking Updates

Track changes to salary information and record old and new values:

CREATE TRIGGER trgAfterUpdate
ON Employees
AFTER UPDATE
AS
BEGIN
    INSERT INTO AuditLog (Action, EmployeeID, ActionDate, OldSalary, NewSalary)
    SELECT 'UPDATE', i.EmployeeID, GETDATE(), d.Salary, i.Salary
    FROM inserted i
    JOIN deleted d ON i.EmployeeID = d.EmployeeID;
END;

Archiving Deletes

Archive deleted employee records to another table:

CREATE TRIGGER trgAfterDelete
ON Employees
AFTER DELETE
AS
BEGIN
    INSERT INTO DeletedEmployees (EmployeeID, Name, TerminationDate)
    SELECT EmployeeID, Name, GETDATE()
    FROM deleted;
END;

Summary

Triggering events are critical in SQL triggers as they define when and why the trigger should execute. Understanding how to create triggers that respond to INSERT, UPDATE, and DELETE events allows for powerful automation and data management within your database. Learning how to implement and manage these triggers effectively is essential for robust database design and operational integrity.
