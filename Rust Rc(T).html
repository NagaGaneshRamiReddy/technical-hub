Rust Rc <T>

Understanding Rc <T>

Rc <T>, short for "Reference Counted", is a smart pointer in Rust that enables multiple ownership of the same data. It keeps track of the number of references to a value, allowing safe sharing of data without copying it.

Key Features of Rc <T>

  • Multiple Ownership: Allows multiple owners of the same data.
  • Immutable by Default: Data stored in Rc cannot be modified through the reference count.
  • Non-thread Safe: Rc <T> is not safe for concurrent use; use Arc <T> for thread-safe scenarios.

Using Rc <T> with Structs

You can also use Rc <T> to enable shared ownership of complex data structures, such as structs:

use std::rc::Rc;
            
            struct Node {
                value: i32,
                next: Option <Rc <Node>>,
            }
           
            fn main() {
                let node1 = Rc::new(Node {
                    value: 1,
                    next: None,
                });
                
                let node2 = Rc::new(Node {
                    value: 2,
                    next: Some(Rc::clone(&node1)),
                });
            
                println!("Node 2 value: {}", node2.value);
                println!("Node 1 value from Node 2: {}", node2.next.as_ref().unwrap().value);
            }

Summary

Rc<T> is a powerful tool for managing shared ownership of data in Rust. By enabling multiple references to the same value, it simplifies memory management while maintaining safety and preventing data races.
