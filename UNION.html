UNION

SQL UNION

The UNION operation in SQL is used to combine the results of two or more SELECT queries into a single result set. It removes duplicate rows between the combined queries by default. If you want to include duplicate rows, you can use the UNION ALL operation.

Basic Syntax

    SELECT columns
    FROM table1
    UNION
    SELECT columns
    FROM table2;
  • columns - The columns you want to retrieve from each table. The columns must be of the same data type and in the same order in each SELECT statement.
  • table1 - The first table in the query.
  • table2 - The second table in the query.

Examples of UNION

1. Basic UNION

Retrieve a list of all employee names from two different departments:

    SELECT emp_name
    FROM employees_department1
    UNION
    SELECT emp_name
    FROM employees_department2;

This query combines the employee names from employees_department1 and employees_department2. Duplicate names are removed from the result set.

2. UNION ALL

Retrieve a list of all employee names from two departments, including duplicates:

    SELECT emp_name
    FROM employees_department1
    UNION ALL
    SELECT emp_name
    FROM employees_department2;

This query combines the employee names from employees_department1 and employees_department2, including any duplicate names.

3. UNION with Different Columns

Retrieve a combined list of employees and contractors, where each has a name and an identifier:

    SELECT emp_name, emp_id AS id
    FROM employees
    UNION
    SELECT contractor_name, contractor_id AS id
    FROM contractors;

In this query, the employees and contractors tables are combined. The columns are aligned and renamed to ensure compatibility.

Advanced Usage of UNION

1. UNION with Sorting

Retrieve a combined list of employee and contractor names, sorted alphabetically:

    SELECT emp_name
    FROM employees
    UNION
    SELECT contractor_name
    FROM contractors
    ORDER BY emp_name;

This query combines names from both the employees and contractors tables and sorts the result alphabetically.

2. UNION with Aggregation

Retrieve the total number of employees and contractors combined:

    SELECT COUNT(*) AS total_count
    FROM (
        SELECT emp_name FROM employees
        UNION
        SELECT contractor_name FROM contractors
    ) AS combined;

This query combines the names from employees and contractors tables and counts the total number of unique names.

Summary

The UNION operation in SQL is used to combine the results of multiple SELECT queries into a single result set, removing duplicates by default. Use UNION ALL if you need to include duplicates. Ensure that the SELECT statements have the same number of columns and compatible data types for the operation to work correctly.
