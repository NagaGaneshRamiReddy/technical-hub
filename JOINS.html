JOINS

SQL JOINs

JOINs are used in SQL to combine rows from two or more tables based on a related column. This allows you to retrieve and work with data from multiple tables in a single query. JOINs are essential for querying relational databases where data is distributed across different tables.

Types of JOINs

1. INNER JOIN

The INNER JOIN keyword selects records that have matching values in both tables involved in the join. It is the most common type of join.

    SELECT table1.column1, table2.column2
    FROM table1
    INNER JOIN table2 ON table1.common_column = table2.common_column;

This query retrieves records where there is a match between table1 and table2 based on the common_column.

2. LEFT JOIN (or LEFT OUTER JOIN)

The LEFT JOIN keyword returns all records from the left table and the matched records from the right table. If there is no match, NULL values are returned for columns from the right table.

    SELECT table1.column1, table2.column2
    FROM table1
    LEFT JOIN table2 ON table1.common_column = table2.common_column;

This query retrieves all records from table1 and the matching records from table2. If there is no match, NULL values will be shown for table2 columns.

3. RIGHT JOIN (or RIGHT OUTER JOIN)

The RIGHT JOIN keyword returns all records from the right table and the matched records from the left table. If there is no match, NULL values are returned for columns from the left table.

    SELECT table1.column1, table2.column2
    FROM table1
    RIGHT JOIN table2 ON table1.common_column = table2.common_column;

This query retrieves all records from table2 and the matching records from table1. If there is no match, NULL values will be shown for table1 columns.

4. FULL JOIN (or FULL OUTER JOIN)

The FULL JOIN keyword returns all records when there is a match in either left or right table. If there is no match, NULL values are returned for columns from the table without a match.

    SELECT table1.column1, table2.column2
    FROM table1
    FULL JOIN table2 ON table1.common_column = table2.common_column;

This query retrieves all records from both table1 and table2. If there is no match, NULL values will be shown for columns from the table that does not have a corresponding record.

5. CROSS JOIN

The CROSS JOIN keyword returns the Cartesian product of both tables, i.e., all possible combinations of rows from the tables. This join does not require a condition.

    SELECT table1.column1, table2.column2
    FROM table1
    CROSS JOIN table2;

This query returns all possible combinations of rows between table1 and table2.

6. SELF JOIN

A SELF JOIN is a join in which a table is joined with itself. This can be useful for querying hierarchical data or comparing rows within the same table.

    SELECT a.column1, b.column1
    FROM table a
    INNER JOIN table b ON a.common_column = b.common_column
    WHERE a.condition_column = 'some_value';

This query compares rows within the same table by joining it with itself, based on a common column and specified conditions.

Examples of JOINs

Example 1: INNER JOIN

    SELECT employees.emp_name, departments.dept_name
    FROM employees
    INNER JOIN departments ON employees.dept_id = departments.dept_id;

This query retrieves the names of employees and their corresponding department names by joining the employees and departments tables on the dept_id column.

Example 2: LEFT JOIN

    SELECT students.student_name, courses.course_name
    FROM students
    LEFT JOIN enrollments ON students.student_id = enrollments.student_id
    LEFT JOIN courses ON enrollments.course_id = courses.course_id;

This query retrieves all student names and the names of the courses they are enrolled in. If a student is not enrolled in any course, NULL values will be shown for course_name.

Summary

JOINs are a fundamental aspect of SQL, allowing you to combine and query data from multiple tables based on relationships between them. Understanding different types of JOINs and their usage is crucial for effective database querying and management.
