Generics in Java

Introduction to Generics

Generics allow you to define classes, interfaces, and methods with type parameters. They provide a way to create a single class or method that can work with different types of data, improving code reusability and type safety. Generics help ensure that errors are caught at compile time rather than at runtime.

Why Use Generics?

  • Type Safety: Generics ensure that you only use the intended types, reducing runtime errors related to type mismatches.
  • Code Reusability: Generics allow you to write code that can work with any type, avoiding code duplication and improving maintainability.
  • Elimination of Type Casting: With Generics, you do not need to cast objects when retrieving them from collections, making the code cleaner and less error-prone.

Generic Classes

A generic class is a class that can work with any type of data. You define a class with one or more type parameters, which are specified in angle brackets. Here’s an example:

public class Box<T> {
    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }
}

public class Main {
    public static void main(String[] args) {
        // Create a Box for Integer
        Box<Integer> integerBox = new Box<>();
        integerBox.set(123);
        System.out.println("Integer value: " + integerBox.get());

        // Create a Box for String
        Box<String> stringBox = new Box<>();
        stringBox.set("Hello Generics");
        System.out.println("String value: " + stringBox.get());
    }
}

Generic Methods

Generic methods are methods that can operate on different types of data. The type parameter is specified before the return type of the method. Here’s an example:

public class GenericMethodExample {
    public <T> void printArray(T[] array) {
        for (T element : array) {
            System.out.println(element);
        }
    }

    public static void main(String[] args) {
        GenericMethodExample example = new GenericMethodExample();

        // Print Integer array
        Integer[] intArray = {1, 2, 3, 4, 5};
        example.printArray(intArray);

        // Print String array
        String[] strArray = {"A", "B", "C"};
        example.printArray(strArray);
    }
}

Generic Interfaces

Generic interfaces are interfaces with type parameters. They can be implemented by classes with specific types. Here’s an example:

public interface Pair<K, V> {
    K getKey();
    V getValue();
}

public class OrderedPair<K, V> implements Pair<K, V> {
    private K key;
    private V value;

    public OrderedPair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() {
        return key;
    }

    public V getValue() {
        return value;
    }
}

public class Main {
    public static void main(String[] args) {
        // Create an OrderedPair
        Pair<String, Integer> pair = new OrderedPair<>("Age", 30);
        System.out.println("Key: " + pair.getKey());
        System.out.println("Value: " + pair.getValue());
    }
}

Bounded Type Parameters

Bounded type parameters allow you to restrict the types that can be used as type arguments. You can specify that a type parameter must be a subclass of a particular class or implement a specific interface:

public <T extends Number> void printNumber(T number) {
    System.out.println("Number: " + number);
}

public class Main {
    public static void main(String[] args) {
        GenericsExample example = new GenericsExample();

        // Print Integer
        example.printNumber(123);

        // Print Double
        example.printNumber(123.45);
    }
}

Wildcards

Wildcards allow you to specify an unknown type. They are used in situations where you want to work with different types without specifying the exact type:

  • Unbounded Wildcard: ? can be used to represent any type.
  • Bounded Wildcards: Use ? extends T for upper bounds (i.e., a type that is a subclass of T) and ? super T for lower bounds (i.e., a type that is a superclass of T).
public void processList(List<? extends Number> list) {
    for (Number number : list) {
        System.out.println(number);
    }
}

Summary

Generics in Java enhance type safety and code reusability by allowing you to define classes, interfaces, and methods with type parameters. They provide a way to create flexible and type-safe code that can work with different data types. Understanding generics, including generic classes, methods, interfaces, bounded type parameters, and wildcards, is crucial for writing effective and maintainable Java code.
