FULL JOIN

SQL FULL JOIN

The FULL JOIN (or FULL OUTER JOIN) operation in SQL is used to combine rows from two tables based on a related column, returning all rows when there is a match in either table. If there is no match, NULL values are returned for columns from the table that does not have a match.

Basic Syntax

    SELECT columns
    FROM table1
    FULL JOIN table2
    ON table1.common_column = table2.common_column;
  • columns - The columns you want to retrieve from both tables.
  • table1 - The first table in the join operation.
  • table2 - The second table in the join operation.
  • common_column - The column used to match rows between the two tables.

Examples of FULL JOIN

1. Basic FULL JOIN

Retrieve a list of all employees and departments, including those without a match:

    SELECT employees.emp_name, departments.dept_name
    FROM employees
    FULL JOIN departments
    ON employees.dept_id = departments.dept_id;

This query joins the employees table with the departments table. It returns all employees and all departments, including those that do not have a matching department or employee, respectively. Non-matching rows from either table will have NULL values in the columns from the other table.

2. FULL JOIN with Filtering

Retrieve all employees and departments, but only include results where the department name is 'HR' or the employee's name is not NULL:

    SELECT employees.emp_name, departments.dept_name
    FROM employees
    FULL JOIN departments
    ON employees.dept_id = departments.dept_id
    WHERE departments.dept_name = 'HR' OR employees.emp_name IS NOT NULL;

This query includes all departments named 'HR' and all employees, showing NULL values where there are no corresponding matches.

3. FULL JOIN with Aggregation

Retrieve a count of employees and departments, including those without employees:

    SELECT departments.dept_name, COUNT(employees.emp_id) AS num_employees
    FROM departments
    FULL JOIN employees
    ON departments.dept_id = employees.dept_id
    GROUP BY departments.dept_name;

This query joins the departments and employees tables and counts the number of employees in each department. Departments without employees will have a count of zero.

Advanced Usage of FULL JOIN

1. FULL JOIN with Multiple Tables

Retrieve a list of students, their courses, and their instructors, including students and courses without a match:

    SELECT students.student_name, courses.course_name, instructors.instructor_name
    FROM students
    FULL JOIN enrollments ON students.student_id = enrollments.student_id
    FULL JOIN courses ON enrollments.course_id = courses.course_id
    FULL JOIN instructors ON courses.instructor_id = instructors.instructor_id;

This query joins the students, enrollments, courses, and instructors tables. It returns all students, all courses, and all instructors, including those without any matches.

2. FULL JOIN with Sorting

Retrieve all departments and their employees, sorted by department names and employee names:

    SELECT employees.emp_name, departments.dept_name
    FROM employees
    FULL JOIN departments
    ON employees.dept_id = departments.dept_id
    ORDER BY departments.dept_name, employees.emp_name;

This query retrieves all departments and employees, sorting the results by department names and then by employee names.

Summary

The FULL JOIN is a comprehensive SQL operation that returns all rows from both tables, including rows that do not have matching values. This join ensures that no data is omitted from either table, providing a complete view of the combined data.
