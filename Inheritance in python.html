Inheritance in Python

What is Inheritance?

Inheritance is a fundamental concept in object-oriented programming that allows one class (the child or subclass) to inherit attributes and methods from another class (the parent or superclass). This promotes code reuse and establishes a hierarchical relationship between classes.

Types of Inheritance

  • Single Inheritance: A class inherits from a single parent class.
  • Multiple Inheritance: A class inherits from more than one parent class.
  • Multilevel Inheritance: A class inherits from a class that is already a subclass of another class.
  • Hierarchical Inheritance: Multiple classes inherit from a single parent class.
  • Hybrid Inheritance: A combination of two or more types of inheritance.

Syntax and Examples

Single Inheritance

In single inheritance, a class inherits from one parent class:

# Define the parent class
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} makes a sound."

# Define the child class inheriting from Animal
class Dog(Animal):
    def speak(self):
        return f"{self.name} barks."

# Create an object of the child class
dog = Dog(name="Buddy")
print(dog.speak())  # Output: Buddy barks.

Multiple Inheritance

In multiple inheritance, a class inherits from more than one parent class:

# Define the first parent class
class Flyer:
    def fly(self):
        return "Can fly."

# Define the second parent class
class Swimmer:
    def swim(self):
        return "Can swim."

# Define the child class inheriting from Flyer and Swimmer
class Duck(Flyer, Swimmer):
    def speak(self):
        return "Quack."

# Create an object of the child class
duck = Duck()
print(duck.fly())  # Output: Can fly.
print(duck.swim())  # Output: Can swim.
print(duck.speak())  # Output: Quack.

Multilevel Inheritance

In multilevel inheritance, a class inherits from a class that is already a subclass of another class:

# Define the base class
class Vehicle:
    def has_wheels(self):
        return "Has wheels."

# Define the intermediate class inheriting from Vehicle
class Car(Vehicle):
    def has_doors(self):
        return "Has doors."

# Define the final class inheriting from Car
class ElectricCar(Car):
    def has_battery(self):
        return "Has a battery."

# Create an object of the final class
e_car = ElectricCar()
print(e_car.has_wheels())  # Output: Has wheels.
print(e_car.has_doors())  # Output: Has doors.
print(e_car.has_battery())  # Output: Has a battery.

Hierarchical Inheritance

In hierarchical inheritance, multiple classes inherit from a single parent class:

# Define the parent class
class Parent:
    def common_method(self):
        return "Common method."

# Define the first child class inheriting from Parent
class Child1(Parent):
    def method1(self):
        return "Method from Child1."

# Define the second child class inheriting from Parent
class Child2(Parent):
    def method2(self):
        return "Method from Child2."

# Create objects of the child classes
child1 = Child1()
child2 = Child2()

print(child1.common_method())  # Output: Common method.
print(child1.method1())  # Output: Method from Child1.
print(child2.common_method())  # Output: Common method.
print(child2.method2())  # Output: Method from Child2.

Key Points about Inheritance

  • Code Reuse: Inheritance allows you to reuse code from parent classes, reducing redundancy and improving maintainability.
  • Hierarchy: Establishes a hierarchy between classes, making relationships and functionalities clearer.
  • Method Overriding: Child classes can override methods of the parent class to provide specific functionality.
  • Super Function: The super() function can be used to call methods from a parent class within a child class.

Summary

Inheritance is a key concept in object-oriented programming that allows classes to inherit attributes and methods from other classes. It promotes code reuse, establishes a clear hierarchy, and supports method overriding. Understanding how to use inheritance effectively helps in designing more modular and maintainable code.
