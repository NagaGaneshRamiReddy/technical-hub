What is Ownership in Rust?

Understanding Ownership

Ownership is a key feature of Rust that manages memory safely and efficiently. It ensures that resources are automatically deallocated when they are no longer needed, preventing memory leaks and data races.

The Three Rules of Ownership

Rust’s ownership model is defined by three fundamental rules:

  1. Each value in Rust has a variable that is its owner.
  2. A value can have only one owner at a time.
  3. When the owner of a value goes out of scope, Rust automatically drops the value, freeing the associated memory.

Example of Ownership

Here’s a simple example to illustrate the concept of ownership:

fn main() {
    let s1 = String::from("Hello"); // s1 owns the string
    let s2 = s1; // Ownership is transferred to s2

    // println!("{}", s1); // This line will cause a compile-time error
    println!("{}", s2); // This works, as s2 now owns the string
}

In this example, when s1 is assigned to s2, ownership of the string is moved to s2. After this point, s1 can no longer be used.

Borrowing

To allow multiple variables to access the same data, Rust uses borrowing. Borrowing allows a function to use a variable without taking ownership:

fn main() {
    let s1 = String::from("Hello");
    let len = calculate_length(&s1); // Borrowing s1

    println!("The length of '{}' is {}.", s1, len);
}

fn calculate_length(s: &String) -> usize {
    s.len() // s is a reference to the String
}

Mutable Borrowing

If you want to modify borrowed data, you can create a mutable reference:

fn main() {
    let mut s = String::from("Hello");
    change(&mut s); // Mutable borrowing

    println!("Changed string: {}", s);
}

fn change(s: &mut String) {
    s.push_str(", world!"); // Modifying the original string
}

Summary

Ownership is a core concept in Rust that provides memory safety without the need for a garbage collector. By understanding ownership, borrowing, and references, you can write safe and efficient Rust code.
