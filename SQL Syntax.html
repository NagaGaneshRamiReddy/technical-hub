SQL Syntax

SQL Syntax

SQL (Structured Query Language) uses a standard set of commands and clauses to interact with databases. These commands enable users to create, retrieve, update, and delete data from databases, as well as manage database structure. SQL syntax is relatively straightforward, making it easy for users to perform complex queries and operations on relational databases.

Basic SQL Syntax

  • SQL Statements: SQL commands are written as statements. Each SQL statement typically starts with a keyword like SELECT, INSERT, UPDATE, or DELETE.
  • Case Insensitivity: SQL keywords are not case-sensitive, so SELECT and select are treated the same. However, it's common practice to write keywords in uppercase and table/column names in lowercase.
  • Semicolon: A semicolon (;) is often used to terminate SQL statements, especially when multiple queries are executed at once.

Common SQL Commands

  • SELECT: The SELECT statement is used to retrieve data from one or more tables. It can be combined with other clauses to filter, sort, or aggregate data.
  • INSERT INTO: The INSERT INTO statement is used to insert new rows of data into a table.
  • UPDATE: The UPDATE statement modifies existing data within a table based on specified conditions.
  • DELETE: The DELETE statement removes rows from a table that meet specified conditions.

SQL Syntax Examples

SELECT Statement

The SELECT statement retrieves data from a database. You can specify which columns to retrieve and from which table:


    SELECT column1, column2
    FROM table_name
    WHERE condition;

Example: Retrieve all employees from the "employees" table who work in the "Sales" department:


    SELECT emp_name, emp_id
    FROM employees
    WHERE department = 'Sales';

INSERT INTO Statement

The INSERT INTO statement adds new rows of data to a table:


    INSERT INTO table_name (column1, column2, column3)
    VALUES (value1, value2, value3);

Example: Insert a new employee into the "employees" table:


    INSERT INTO employees (emp_name, emp_id, department)
    VALUES ('John Doe', 101, 'Marketing');

UPDATE Statement

The UPDATE statement modifies existing data in a table based on a condition:


    UPDATE table_name
    SET column1 = value1, column2 = value2
    WHERE condition;

Example: Update the department of an employee in the "employees" table:


    UPDATE employees
    SET department = 'HR'
    WHERE emp_id = 101;

DELETE Statement

The DELETE statement removes rows from a table that match a specified condition:


    DELETE FROM table_name
    WHERE condition;

Example: Delete an employee from the "employees" table:


    DELETE FROM employees
    WHERE emp_id = 101;

SQL Clauses

SQL queries often include additional clauses to refine or filter data:

  • WHERE: The WHERE clause is used to filter records based on specific conditions.
  • ORDER BY: The ORDER BY clause sorts the result set by one or more columns in ascending or descending order.
  • GROUP BY: The GROUP BY clause groups rows that have the same values into summary rows, often used with aggregate functions like COUNT, SUM, AVG, etc.
  • HAVING: The HAVING clause filters rows after the GROUP BY clause has been applied.

Summary

SQL syntax provides a powerful yet easy-to-use way to interact with relational databases. By mastering basic SQL commands like SELECT, INSERT, UPDATE, and DELETE, users can retrieve and manipulate data efficiently. Clauses such as WHERE, ORDER BY, and GROUP BY allow for fine-tuned control over data queries, making SQL a versatile tool in data management and analysis.
