Recoverable Errors

Understanding Recoverable Errors in Rust

Recoverable errors in Rust are situations where the program can continue running even if an error occurs. These are typically handled using the Result type, which allows functions to return either a successful result or an error.

The Result Type

The Result type is defined as follows:

enum Result {
    Ok(T),   // Represents a successful result
    Err(E),  // Represents an error
}

Here, T is the type of the successful value, and E is the type of the error.

Example of Using Result

Hereâ€™s an example function that performs division and returns a Result:

fn divide(a: f64, b: f64) -> Result {
    if b == 0.0 {
        Err(String::from("Cannot divide by zero"))
    } else {
        Ok(a / b)
    }
}

fn main() {
    match divide(10.0, 0.0) {
        Ok(result) => println!("Result: {}", result),
        Err(e) => println!("Error: {}", e), // Outputs: Error: Cannot divide by zero
    }
}

Using the ? Operator

The ? operator can be used to simplify error handling in functions that return a Result. It automatically returns the error if it occurs:

fn get_file_content(file_path: &str) -> Result {
    let content = std::fs::read_to_string(file_path)?;
    Ok(content)
}

fn main() {
    match get_file_content("file.txt") {
        Ok(content) => println!("File content: {}", content),
        Err(e) => println!("Error reading file: {}", e),
    }
}

Custom Error Types

You can create custom error types to provide more context for errors. Implement the std::error::Error trait for better integration with Rustâ€™s error handling system:

#[derive(Debug)]
struct MyError(String);

impl std::fmt::Display for MyError {
    fn fmt(&self, f: &mut std::fmt::Formatter) -> std::fmt::Result {
        write!(f, "MyError: {}", self.0)
    }
}

impl std::error::Error for MyError {}

fn might_fail() -> Result<(), MyError> {
    Err(MyError(String::from("An error occurred")))
}

fn main() {
    match might_fail() {
        Ok(_) => println!("Success!"),
        Err(e) => println!("Error: {}", e),
    }
}
