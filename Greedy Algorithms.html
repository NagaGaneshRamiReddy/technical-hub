Greedy algorithms

Introduction

Greedy algorithms are a class of algorithms that make locally optimal choices at each step with the hope of finding a global optimum. Unlike Dynamic Programming, which solves problems by breaking them down into overlapping subproblems, Greedy Algorithms work by making a series of choices, each of which looks best at the moment.

Principles of Greedy Algorithms

Greedy algorithms are based on the following principles:

  • Greedy Choice Property: A global optimum can be arrived at by selecting a local optimum.
  • Optimal Substructure: An optimal solution to the problem contains optimal solutions to subproblems.

Types of Greedy Algorithms

Greedy algorithms can be categorized based on the nature of the problems they solve. Some common types include:

  • Activity Selection Problem: Selecting the maximum number of activities that don't overlap.
  • Kruskal's Algorithm: Finding the Minimum Spanning Tree (MST) of a graph.
  • Dijkstra's Algorithm: Finding the shortest path in a weighted graph.
  • Huffman Coding: Compressing data using variable-length codes based on frequency.

Examples

1. Activity Selection Problem

The goal is to select the maximum number of activities that can be performed by a single person, given their start and end times. Activities are selected based on their finish times, aiming to maximize the number of non-overlapping activities.


def activity_selection(start, end):
    n = len(start)
    selected = []

    # Sort activities based on their end times
    activities = sorted(range(n), key=lambda i: end[i])

    last_end = -1
    for i in activities:
        if start[i] > last_end:
            selected.append(i)
            last_end = end[i]

    return selected

# Example usage
start = [1, 3, 0, 5, 8, 5]
end = [2, 4, 6, 7, 9, 9]
print("Selected activities:", activity_selection(start, end))

2. Kruskal's Algorithm

Kruskal's Algorithm finds the Minimum Spanning Tree (MST) of a graph by sorting edges based on weight and adding them to the MST while avoiding cycles.


class DisjointSet:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, u):
        if self.parent[u] != u:
            self.parent[u] = self.find(self.parent[u])
        return self.parent[u]

    def union(self, u, v):
        root_u = self.find(u)
        root_v = self.find(v)
        if root_u != root_v:
            if self.rank[root_u] > self.rank[root_v]:
                self.parent[root_v] = root_u
            elif self.rank[root_u] < self.rank[root_v]:
                self.parent[root_u] = root_v
            else:
                self.parent[root_v] = root_u
                self.rank[root_u] += 1

def kruskal(n, edges):
    mst = []
    edges = sorted(edges, key=lambda e: e[2])
    ds = DisjointSet(n)

    for u, v, weight in edges:
        if ds.find(u) != ds.find(v):
            ds.union(u, v)
            mst.append((u, v, weight))

    return mst

# Example usage
edges = [
    (0, 1, 10), (0, 2, 6), (0, 3, 5),
    (1, 3, 15), (2, 3, 4)
]
n = 4
print("Minimum Spanning Tree:", kruskal(n, edges))

Summary

Greedy algorithms are a powerful approach to problem-solving where local optimization leads to a global solution. By making a series of choices that seem best at each step, greedy algorithms can efficiently solve problems like activity selection, graph-based problems, and more. Mastering these algorithms provides valuable insights into optimization and efficient computation in various applications.
