Multithreading and Concurrency

Introduction to Multithreading

Multithreading is a Java feature that allows multiple threads to run concurrently within a single program. Each thread represents a separate path of execution, enabling tasks to be performed in parallel, which can improve performance and responsiveness in applications.

Creating Threads

In Java, you can create threads in two primary ways:

  • By Extending the Thread Class: Create a new class that extends the Thread class and override its run() method.
  • By Implementing the Runnable Interface: Implement the Runnable interface and pass an instance of your class to a Thread object.

Extending the Thread Class

public class MyThread extends Thread {
    public void run() {
        System.out.println("Thread is running");
    }

    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.start(); // Starts the thread
    }
}

Implementing the Runnable Interface

public class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Runnable is running");
    }

    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();
        Thread thread = new Thread(myRunnable);
        thread.start(); // Starts the thread
    }
}

Thread Lifecycle

Threads in Java go through several states during their lifecycle:

  • New: A thread that has been created but not yet started.
  • Runnable: A thread that is eligible to run but not yet running.
  • Blocked: A thread that is waiting for a resource or condition.
  • Waiting: A thread that is waiting indefinitely for another thread to perform a particular action.
  • Timed Waiting: A thread that is waiting for a specified period.
  • Terminated: A thread that has completed execution.

Thread Synchronization

Synchronization is crucial when multiple threads access shared resources to avoid data inconsistency and ensure thread safety. You can synchronize methods or blocks of code to control access:

public class SynchronizedExample {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public static void main(String[] args) {
        SynchronizedExample example = new SynchronizedExample();
        
        Thread t1 = new Thread(() -> example.increment());
        Thread t2 = new Thread(() -> example.increment());

        t1.start();
        t2.start();
    }
}

Concurrency Utilities

Java provides several concurrency utilities in the java.util.concurrent package to simplify multithreading and synchronization:

  • Executor Framework: Manages a pool of threads and handles task execution. For example, ExecutorService allows you to submit tasks and manage thread pools.
  • Concurrent Collections: Provide thread-safe implementations of common data structures, such as ConcurrentHashMap and CopyOnWriteArrayList.
  • Locks: More flexible and advanced synchronization than synchronized blocks, such as ReentrantLock.
  • Atomic Variables: Provide thread-safe operations for single variables, like AtomicInteger and AtomicReference.

Using ExecutorService

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ExecutorServiceExample {
    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(2);

        executor.submit(() -> System.out.println("Task 1"));
        executor.submit(() -> System.out.println("Task 2"));

        executor.shutdown();
    }
}

Example: Multithreading and Synchronization

Here’s a complete example demonstrating multithreading, synchronization, and the use of concurrency utilities:

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.atomic.AtomicInteger;

public class MultithreadingExample {
    private static AtomicInteger count = new AtomicInteger(0);

    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(2);

        Runnable task = () -> {
            for (int i = 0; i < 1000; i++) {
                count.incrementAndGet(); // Atomic operation
            }
        };

        executor.submit(task);
        executor.submit(task);

        executor.shutdown();
        while (!executor.isTerminated()) {
            // Wait for all tasks to complete
        }

        System.out.println("Final count: " + count.get());
    }
}

Summary

Multithreading and concurrency in Java enable efficient execution of multiple tasks simultaneously. Understanding how to create and manage threads, synchronize access to shared resources, and use concurrency utilities is essential for developing robust and high-performance Java applications. Mastering these concepts helps in building responsive, scalable, and thread-safe software.
