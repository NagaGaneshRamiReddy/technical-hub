Stored Procedures and Triggers

Stored Procedures and Triggers in SQL

Stored Procedures and Triggers are essential components in SQL that help automate and streamline database operations. Stored procedures are precompiled SQL statements that can be executed as needed, while triggers are automatic actions that occur in response to specific events in the database.

Stored Procedures

Overview

A Stored Procedure is a collection of SQL statements that can be executed as a single unit. Stored procedures are used to encapsulate complex operations, improve performance, and ensure consistency in database interactions.

Creating a Stored Procedure

To create a stored procedure, use the CREATE PROCEDURE statement. Here's an example:

CREATE PROCEDURE GetEmployeeDetails
    @EmployeeID INT
AS
BEGIN
    SELECT * FROM Employees
    WHERE EmployeeID = @EmployeeID;
END;

Executing a Stored Procedure

To execute a stored procedure, use the EXEC statement:

EXEC GetEmployeeDetails @EmployeeID = 1;

Updating a Stored Procedure

To update an existing stored procedure, use the ALTER PROCEDURE statement:

ALTER PROCEDURE GetEmployeeDetails
    @EmployeeID INT
AS
BEGIN
    SELECT Name, Position FROM Employees
    WHERE EmployeeID = @EmployeeID;
END;

Dropping a Stored Procedure

To delete a stored procedure, use the DROP PROCEDURE statement:

DROP PROCEDURE GetEmployeeDetails;

Triggers

Overview

A Trigger is a special type of stored procedure that automatically executes in response to certain events on a table, such as insertions, updates, or deletions. Triggers are used to enforce business rules, maintain audit trails, and synchronize tables.

Creating a Trigger

To create a trigger, use the CREATE TRIGGER statement. Here's an example:

CREATE TRIGGER trgAfterInsert
ON Employees
AFTER INSERT
AS
BEGIN
    INSERT INTO AuditLog (Action, EmployeeID, ActionDate)
    SELECT 'INSERT', EmployeeID, GETDATE()
    FROM inserted;
END;

Dropping a Trigger

To delete a trigger, use the DROP TRIGGER statement:

DROP TRIGGER trgAfterInsert;

Examples of Stored Procedures and Triggers

Stored Procedure Example

Retrieve details of employees with a specific position:

CREATE PROCEDURE GetEmployeesByPosition
    @Position VARCHAR(50)
AS
BEGIN
    SELECT * FROM Employees
    WHERE Position = @Position;
END;

Trigger Example

Automatically log changes to employee records:

CREATE TRIGGER trgAfterUpdate
ON Employees
AFTER UPDATE
AS
BEGIN
    INSERT INTO AuditLog (Action, EmployeeID, ActionDate)
    SELECT 'UPDATE', EmployeeID, GETDATE()
    FROM inserted;
END;

Summary

Stored Procedures and Triggers are powerful tools in SQL for automating and managing database operations. Stored procedures allow you to encapsulate complex logic and improve performance, while triggers provide automatic responses to data changes, ensuring data integrity and consistency. Mastery of these features can greatly enhance your ability to manage and optimize your SQL databases.
