Advanced C++ concepts

Introduction

Advanced C++ concepts build on the fundamentals of the language, introducing more complex features and techniques that enhance your ability to write efficient and powerful code. These concepts include advanced memory management, metaprogramming, concurrency, and more. Understanding these topics is crucial for mastering C++ and tackling complex programming challenges.

1. Memory Management

C++ offers fine-grained control over memory management, which includes dynamic memory allocation and deallocation:

  • Smart Pointers: Use std::unique_ptr, std::shared_ptr, and std::weak_ptr from the C++ Standard Library to manage dynamic memory automatically and prevent memory leaks.
  • Manual Memory Management: Allocate and deallocate memory using new and delete operators. Be cautious to avoid memory leaks and dangling pointers.
#include <iostream>
#include <memory>

void demoSmartPointer() {
    std::unique_ptr<int> ptr1 = std::make_unique<int>(10);
    std::cout << "Value: " << *ptr1 << std::endl;
    
    std::shared_ptr<int> ptr2 = std::make_shared<int>(20);
    std::cout << "Value: " << *ptr2 << std::endl;
}

2. Metaprogramming

Metaprogramming allows you to write programs that generate or manipulate other programs at compile-time:

  • Template Metaprogramming: Use template classes and functions to perform computations at compile-time. This includes techniques like SFINAE (Substitution Failure Is Not An Error) and type traits.
  • Constant Expressions: Use constexpr functions to compute values at compile-time.
#include <iostream>

// Compile-time factorial calculation
constexpr int factorial(int n) {
    return (n > 1) ? n * factorial(n - 1) : 1;
}

int main() {
    std::cout << "Factorial of 5: " << factorial(5) << std::endl;
    return 0;
}

3. Concurrency

Concurrency in C++ involves executing multiple threads simultaneously to improve performance:

  • Threads: Use the std::thread class to create and manage threads.
  • Synchronization: Use synchronization mechanisms like std::mutex, std::lock_guard, and std::condition_variable to manage shared resources and avoid race conditions.
#include <iostream>
#include <thread>
#include <mutex>

std::mutex mtx;

void printNumbers() {
    std::lock_guard<std::mutex> lock(mtx);
    for (int i = 0; i < 5; ++i) {
        std::cout << i << " ";
    }
    std::cout << std::endl;
}

int main() {
    std::thread t1(printNumbers);
    std::thread t2(printNumbers);
    t1.join();
    t2.join();
    return 0;
}

4. Advanced Template Techniques

Advanced template techniques include:

  • Variadic Templates: Templates that accept a variable number of template arguments.
  • CRTP (Curiously Recurring Template Pattern): A design pattern where a class template inherits from a base template instantiation.
#include <iostream>

template <typename T>
class Base {
public:
    void show() {
        static_cast<T*>(this)->print();
    }
};

class Derived : public Base<Derived> {
public:
    void print() {
        std::cout << "Derived class" << std::endl;
    }
};

int main() {
    Derived d;
    d.show();
    return 0;
}

5. Lambda Expressions

Lambda expressions provide a concise way to define anonymous functions and are useful for short-term operations:

#include <iostream>

int main() {
    auto lambda = [](int a, int b) { return a + b; };
    std::cout << "Sum: " << lambda(5, 3) << std::endl;
    return 0;
}
