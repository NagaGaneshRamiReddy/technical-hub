Rust Modules

Understanding Modules in Rust

A module in Rust is a way to organize your code into separate namespaces, allowing you to manage complexity in larger codebases. Modules provide encapsulation, helping to control the visibility of functions and types.

Defining a Module

You define a module using the mod keyword followed by the module's name:

mod my_module {
            fn say_hello() {
                println!("Hello from the module!");
            }
        }

In this example, we've created a module named my_module containing a function called say_hello.

Accessing Module Functions

To access functions or items within a module, use the syntax module_name::item_name:

fn main() {
            my_module::say_hello();
        }
        
        mod my_module {
            pub fn say_hello() {
                println!("Hello from the module!");
            }
        }

Here, the say_hello function is marked as pub (public) so it can be accessed from outside the module.

Organizing Modules Across Files

Rust allows you to organize modules across multiple files. By default, the Rust compiler expects a module's code to be in the same file or in a separate file named after the module.

  • main.rs (Main file)
  • my_module.rs (Separate file for the module)

Example of a Module in Another File

Hereâ€™s an example of a module defined in a separate file:

// main.rs
        mod my_module;
        
        fn main() {
            my_module::say_hello();
        }
// my_module.rs
        pub fn say_hello() {
            println!("Hello from the separate module file!");
        }

Re-exporting with "pub use"

You can re-export items from one module to another using pub use. This simplifies access to nested modules:

mod my_module {
            pub mod nested_module {
                pub fn greet() {
                    println!("Hello from the nested module!");
                }
            }
        }
        
        // Re-export the function for easier access
        pub use my_module::nested_module::greet;
        
        fn main() {
            greet(); // Call greet directly
        }

Modules in Rust help you organize your code and manage its visibility, leading to cleaner and more maintainable code. By effectively using modules, you can structure your projects in a scalable way.
