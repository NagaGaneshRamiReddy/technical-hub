Rust Error Handling

Understanding Error Handling in Rust

Error handling is a crucial aspect of Rust, which emphasizes safety and robustness. Rust provides two main types of error handling: panic for unrecoverable errors and Result for recoverable errors.

Panic: Unrecoverable Errors

When a panic occurs, the program will stop execution and print an error message. This is typically used for situations where the program cannot recover:

fn main() {
    let x = vec![1, 2, 3];
    // This will cause a panic if the index is out of bounds
    let fourth_element = &x[3]; // Index out of bounds
    println!("{}", fourth_element);
}

Result: Recoverable Errors

The Result type is used for functions that can return an error. It is an enum that can be either Ok for success or Err for failure:

fn divide(a: f64, b: f64) -> Result {
    if b == 0.0 {
        Err(String::from("Cannot divide by zero"))
    } else {
        Ok(a / b)
    }
}

fn main() {
    match divide(10.0, 2.0) {
        Ok(result) => println!("Result: {}", result), // Outputs: Result: 5
        Err(e) => println!("Error: {}", e),
    }
}

Using the ? Operator

The ? operator is a convenient way to propagate errors in functions that return a Result. If the result is Err, it will return early with the error:

fn get_file_content(file_path: &str) -> Result {
    let content = std::fs::read_to_string(file_path)?;
    Ok(content)
}

fn main() {
    match get_file_content("file.txt") {
        Ok(content) => println!("File content: {}", content),
        Err(e) => println!("Error reading file: {}", e),
    }
}

Custom Error Types

You can define your own error types by implementing the std::error::Error trait. This allows for more descriptive error handling:

#[derive(Debug)]
struct MyError;

impl std::fmt::Display for MyError {
    fn fmt(&self, f: &mut std::fmt::Formatter) -> std::fmt::Result {
        write!(f, "My custom error occurred")
    }
}

impl std::error::Error for MyError {}

fn might_fail() -> Result<(), MyError> {
    Err(MyError)
}

fn main() {
    match might_fail() {
        Ok(_) => println!("Success!"),
        Err(e) => println!("Error: {}", e),
    }
}

Summary

Rust's error handling model encourages developers to handle errors explicitly and safely. Using panic! for unrecoverable errors and the Result type for recoverable errors allows for robust and clear error management.
