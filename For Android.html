Publishing Flutter App for Android

Introduction

Publishing a Flutter app on Android involves a series of steps, including app configuration, signing, building the release version, and uploading to the Google Play Store. This guide outlines the entire process.

Step 1: Update the App Version

Before building the release version of your app, ensure that the version number and build number are properly configured in the android/app/build.gradle file. These numbers help you keep track of updates to your app:

defaultConfig {
    versionCode 1 // Increment for each release
    versionName "1.0.0" // Visible version number for users
}

Step 2: Add the App Icon

Your app icon is critical for branding and should be placed in the appropriate resolution folders under android/app/src/main/res. You can use tools like appicon.co to generate icons for all necessary sizes.

Ensure that you place the icons in these folders:

  • mipmap-mdpi
  • mipmap-hdpi
  • mipmap-xhdpi
  • mipmap-xxhdpi
  • mipmap-xxxhdpi

Step 3: Configure App Signing

To publish an Android app, you must sign it with a keystore file. If you don't already have a keystore, you can create one using the following command:

keytool -genkey -v -keystore ~/my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias

This command will generate a keystore file, which you will use to sign the app. Make sure to store this file in a safe location as you will need it for future updates to your app.

Configure key.properties

Create a file named key.properties in the android folder and add your keystore configuration:

storePassword=
keyPassword=
keyAlias=my-key-alias
storeFile=../my-release-key.jks

Update build.gradle

In the android/app/build.gradle file, configure the release signing configuration by adding the following:

android {
    ...
    signingConfigs {
        release {
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
            storeFile file(keystoreProperties['storeFile'])
            storePassword keystoreProperties['storePassword']
        }
    }
    buildTypes {
        release {
            minifyEnabled false // Remove this line if you want code obfuscation
            signingConfig signingConfigs.release
        }
    }
}

Step 4: Build the Release APK or App Bundle

Once your app is configured, you need to generate the release APK or Android App Bundle for distribution. To build the APK, use the following command:

flutter build apk --release

Alternatively, if you want to build an Android App Bundle, which is recommended for uploading to the Google Play Store, use this command:

flutter build appbundle --release

The APK file will be located in the build/app/outputs/flutter-apk directory, and the AAB will be in the build/app/outputs/bundle/release directory.

Step 5: Set Up a Google Play Developer Account

If you haven't already, you'll need to create a Google Play Developer account. There is a one-time registration fee, and once your account is set up, you can publish apps on the Google Play Store.

Go to the Google Play Console and sign in with your Google account. Complete the registration process, and then you can create a new app.

Step 6: Upload Your App to the Google Play Console

After creating your app in the Play Console, you'll need to fill in details such as the app name, description, app category, and contact details. Then, you can upload your APK or app bundle:

  1. Select "Production" from the menu and create a new release.
  2. Upload your signed APK or app bundle file.
  3. Fill out the release notes for your app.

Step 7: Set Up App Details

Once your APK is uploaded, you'll need to provide additional information about your app, including:

  • Content Rating: Use the content rating questionnaire provided by Google.
  • Pricing & Distribution: Set the pricing for your app and select the countries where your app will be available.
  • Data Safety: Declare how your app handles user data and comply with Google Play's data safety requirements.

Step 8: Publish Your App

Once all the required information is complete, you can submit your app for review. The Google Play team will review your app to ensure it complies with their policies and guidelines. After the review process is complete, your app will be published on the Google Play Store.

Conclusion

Publishing a Flutter app for Android involves several important steps, including configuring the app version, signing the APK, and uploading it to the Google Play Console. By following the steps outlined in this guide, you can successfully release your app to the Android market.
