Exception handling in C++

Introduction

Exception handling in C++ is a mechanism to handle runtime errors or exceptions that occur during the execution of a program. By using exceptions, you can separate error-handling code from regular code, making your program more robust and easier to maintain. Exception handling helps in managing unexpected situations and ensuring that your program can handle errors gracefully.

Basic Syntax

Exception handling in C++ involves three main components:

  • try: This block contains the code that might throw an exception.
  • catch: This block contains the code that handles the exception.
  • throw: This keyword is used to signal the occurrence of an exception.
#include <iostream>

int main() {
    try {
        // Code that may throw an exception
        throw std::runtime_error("An error occurred");
    } catch (const std::exception &e) {
        // Code to handle the exception
        std::cout << "Exception caught: " << e.what() << std::endl;
    }
    return 0;
}

Throwing Exceptions

You can throw exceptions using the throw keyword. It is often used to indicate errors or unusual conditions:

#include <iostream>

void divide(int a, int b) {
    if (b == 0) {
        throw std::invalid_argument("Division by zero is not allowed");
    }
    std::cout << "Result: " << (a / b) << std::endl;
}

int main() {
    try {
        divide(10, 0);
    } catch (const std::invalid_argument &e) {
        std::cout << "Exception caught: " << e.what() << std::endl;
    }
    return 0;
}

Handling Exceptions

To handle exceptions, you use the catch block, which follows the try block. The catch block catches exceptions thrown by the try block:

#include <iostream>

int main() {
    try {
        // Code that may throw an exception
        throw std::out_of_range("Out of range error");
    } catch (const std::out_of_range &e) {
        // Code to handle out_of_range exception
        std::cout << "Out of range exception caught: " << e.what() << std::endl;
    } catch (const std::exception &e) {
        // Code to handle any other exception
        std::cout << "Exception caught: " << e.what() << std::endl;
    }
    return 0;
}

Custom Exceptions

You can define your own exception classes to handle specific error conditions:

#include <iostream>
#include <stdexcept>

class MyException : public std::exception {
public:
    const char* what() const noexcept override {
        return "My custom exception occurred";
    }
};

int main() {
    try {
        throw MyException();
    } catch (const MyException &e) {
        std::cout << "Custom exception caught: " << e.what() << std::endl;
    }
    return 0;
}

Exception Safety

Exception safety ensures that your code remains in a valid state even if an exception occurs. There are several levels of exception safety:

  • No-Throw Guarantee: The function guarantees that it will not throw any exceptions.
  • Strong Guarantee: If an exception occurs, the program state is unchanged.
  • Basic Guarantee: The program state is valid, but not necessarily unchanged.

Best Practices

  • Use Exceptions Sparingly: Exceptions should be used for error handling, not for normal control flow.
  • Catch by Reference: Catch exceptions by reference to avoid slicing and improve performance.
  • Be Specific: Catch specific exceptions before general ones to handle different error conditions appropriately.

Summary

Exception handling in C++ is a powerful feature that helps you manage runtime errors and maintain the robustness of your programs. By using try, catch, and throw effectively, you can handle errors gracefully and ensure that your code remains stable. Utilize custom exceptions and follow best practices to enhance the quality of your error handling and overall program design.
