Concurrency in C++

Introduction

Concurrency refers to the ability of a program to execute multiple tasks or threads simultaneously. In C++, concurrency is primarily managed through threads, which can run parallel to each other, allowing for more efficient use of resources and improved performance for certain types of applications. This can be particularly beneficial for tasks that are independent and can be performed concurrently, such as handling multiple client requests or performing complex calculations in parallel.

Threads

In C++, threads are created using the std::thread class, which is part of the Standard Library introduced in C++11. Each thread runs independently and can execute different parts of the code concurrently.

Hereâ€™s how you can create and manage threads in C++:

#include <iostream>
#include <thread>

// Function to be executed by the thread
void print_message() {
    std::cout << "Hello from the thread!" << std::endl;
}

int main() {
    // Create a thread that runs the print_message function
    std::thread t(print_message);

    // Wait for the thread to finish
    t.join();

    std::cout << "Thread has finished executing." << std::endl;

    return 0;
}

In this example:

  • #include <thread> is used to include the thread library.
  • std::thread t(print_message); creates a thread that runs the print_message function.
  • t.join(); waits for the thread to complete before proceeding.

Mutexes

When multiple threads access shared resources, synchronization is needed to prevent data races. This is where std::mutex comes into play. A mutex (short for mutual exclusion) is used to ensure that only one thread can access a critical section of code at a time.

#include <iostream>
#include <thread>
#include <mutex>

std::mutex mtx;  // Mutex for synchronizing access to shared data
int shared_data = 0;

void increment() {
    std::lock_guard<std::mutex> lock(mtx);  // Lock the mutex
    ++shared_data;
    std::cout << "Shared data: " << shared_data << std::endl;
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);

    t1.join();
    t2.join();

    return 0;
}

In this example:

  • #include <mutex> is used to include the mutex library.
  • std::mutex mtx; creates a mutex to synchronize access to shared_data.
  • std::lock_guard<std::mutex> lock(mtx); automatically locks the mutex when entering the critical section and releases it when leaving.

Condition Variables

std::condition_variable is used to synchronize threads based on certain conditions. It allows one thread to wait for a condition to be met and notifies other threads when the condition changes.

#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>

std::mutex mtx;
std::condition_variable cv;
bool ready = false;

void wait_for_signal() {
    std::unique_lock<std::mutex> lock(mtx);
    cv.wait(lock, [] { return ready; });
    std::cout << "Received signal!" << std::endl;
}

void send_signal() {
    std::this_thread::sleep_for(std::chrono::seconds(1));
    {
        std::lock_guard<std::mutex> lock(mtx);
        ready = true;
    }
    cv.notify_one();
}

int main() {
    std::thread t1(wait_for_signal);
    std::thread t2(send_signal);

    t1.join();
    t2.join();

    return 0;
}

In this example:

  • #include <condition_variable> is used to include the condition variable library.
  • cv.wait(lock, [] { return ready; }); makes the thread wait until ready is true.
  • cv.notify_one(); notifies one waiting thread that the condition has changed.
