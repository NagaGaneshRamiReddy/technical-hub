Activity Selection

Introduction

The Activity Selection Problem is a classic example of the Greedy Algorithm approach. The goal is to select the maximum number of non-overlapping activities that can be performed by a single person, given their start and end times. This problem is a useful exercise in understanding greedy algorithms and their application in optimization problems.

Problem Description

Given a set of activities with start and end times, you need to select the activities that can be performed without overlapping. The key is to choose activities that finish early to maximize the number of activities selected.

Greedy Approach

The greedy approach to solving the Activity Selection Problem involves sorting the activities based on their finish times. The algorithm then iterates through the sorted list, selecting activities that start after the last selected activity has finished. This ensures that we are always making a choice that leaves the maximum amount of remaining time for future activities.

Python Implementation


def activity_selection(start, end):
    n = len(start)
    selected = []

    # Sort activities based on their end times
    activities = sorted(range(n), key=lambda i: end[i])

    last_end = -1
    for i in activities:
        if start[i] > last_end:
            selected.append(i)
            last_end = end[i]

    return selected

# Example usage
start = [1, 3, 0, 5, 8, 5]
end = [2, 4, 6, 7, 9, 9]
selected_indices = activity_selection(start, end)
print("Selected activities:", [i for i in selected_indices])

Example

Consider the following activities with their respective start and end times:

  • Activity 1: Start = 1, End = 2
  • Activity 2: Start = 3, End = 4
  • Activity 3: Start = 0, End = 6
  • Activity 4: Start = 5, End = 7
  • Activity 5: Start = 8, End = 9
  • Activity 6: Start = 5, End = 9

Using the greedy algorithm, the selected activities would be:

  • Activity 1
  • Activity 2
  • Activity 4
  • Activity 5

This selection ensures that the maximum number of activities are chosen without overlapping.

Applications

The Activity Selection Problem is relevant in various real-world scenarios, such as:

  • Scheduling: Optimizing schedules for tasks, meetings, or events.
  • Resource Allocation: Allocating limited resources to competing activities.
  • Project Management: Selecting tasks that can be completed within deadlines to maximize productivity.

Summary

The Activity Selection Problem exemplifies how greedy algorithms can be used to optimize the selection of activities based on their timing. By sorting activities by their finish times and iterating through them to select the most fitting ones, this approach ensures the maximum number of activities are selected without overlap. It highlights the effectiveness of greedy algorithms in solving optimization problems efficiently.
