Smart pointers in C++

Introduction

Smart pointers are a type of wrapper around raw pointers that provide automatic memory management and help prevent common issues like memory leaks and dangling pointers. In C++, smart pointers are part of the Standard Library and offer a safer way to handle dynamic memory allocation.

Types of Smart Pointers

C++ provides three main types of smart pointers:

  • std::unique_ptr: Represents exclusive ownership of a resource. Only one std::unique_ptr can own a resource at a time. When the unique_ptr is destroyed, it automatically deletes the resource.
  • std::shared_ptr: Represents shared ownership of a resource. Multiple std::shared_ptr instances can own the same resource. The resource is deleted only when the last shared_ptr owning it is destroyed.
  • std::weak_ptr: A companion to std::shared_ptr that does not affect the reference count. It is used to break circular references and observe shared resources without owning them.

std::unique_ptr

To use std::unique_ptr, include the header <memory>. Here’s a simple example:

#include <iostream>
#include <memory>

int main() {
    std::unique_ptr<int> ptr(new int(10));
    std::cout << "Value: " << *ptr << std::endl;
    
    // No need to explicitly delete the pointer; it is automatically managed
    return 0;
}

In this example:

  • std::unique_ptr<int> ptr(new int(10)); creates a unique pointer managing an integer with the value 10.
  • The memory is automatically released when ptr goes out of scope.

std::shared_ptr

To use std::shared_ptr, include the header <memory>. Here’s an example:

#include <iostream>
#include <memory>

int main() {
    std::shared_ptr<int> ptr1(new int(20));
    std::shared_ptr<int> ptr2 = ptr1;  // Both ptr1 and ptr2 share ownership
    
    std::cout << "Value: " << *ptr1 << std::endl;
    std::cout << "Shared count: " << ptr1.use_count() << std::endl;
    
    // Memory is automatically managed and released when the last shared_ptr is destroyed
    return 0;
}

In this example:

  • std::shared_ptr<int> ptr1(new int(20)); creates a shared pointer managing an integer with the value 20.
  • std::shared_ptr<int> ptr2 = ptr1; creates another shared pointer to the same integer, sharing ownership.
  • ptr1.use_count() returns the number of shared pointers owning the resource.

std::weak_ptr

To use std::weak_ptr, include the header <memory>. Here’s an example:

#include <iostream>
#include <memory>

int main() {
    std::shared_ptr<int> sharedPtr(new int(30));
    std::weak_ptr<int> weakPtr(sharedPtr);
    
    if (auto lockedPtr = weakPtr.lock()) {  // Try to get a shared_ptr from weak_ptr
        std::cout << "Value: " << *lockedPtr << std::endl;
    } else {
        std::cout << "Resource has been released" << std::endl;
    }
    
    return 0;
}

In this example:

  • std::weak_ptr<int> weakPtr(sharedPtr); creates a weak pointer that observes the shared pointer.
  • weakPtr.lock() tries to convert the weak pointer to a shared pointer. If the resource has been deleted, it returns nullptr.

Common Issues and Troubleshooting

When working with smart pointers, be aware of:

  • Cycles in std::shared_ptr: Circular references between shared pointers can lead to memory leaks. Use std::weak_ptr to break cycles.
  • Performance: Using smart pointers involves some overhead compared to raw pointers. Consider this when performance is critical.
