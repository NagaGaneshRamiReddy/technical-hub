JavaScript Interview Questions

1. What is the difference between var, let, and const? ›

var: Declares a function-scoped or globally-scoped variable. It can be redeclared and updated.

let: Declares a block-scoped variable. It can be updated but not redeclared.

const: Declares a block-scoped, read-only constant. It cannot be updated or redeclared.

Example:

var x = 10;
let y = 20;
const z = 30;

x = 15; // valid
y = 25; // valid
z = 35; // Error: Assignment to constant variable.
2. What are JavaScript closures? ›

A closure is a function that remembers the variables from the place where it was defined, even if executed in a different scope.

Example:

function outer() {
    let count = 0;
    return function inner() {
        count++;
        return count;
    }
}

const increment = outer();
console.log(increment()); // 1
console.log(increment()); // 2
3. What is the difference between == and === in JavaScript? ›

== compares two values after converting them to a common type (type coercion), whereas === compares both the values and types (strict equality).

Example:

console.log(1 == '1');  // true (type coercion)
console.log(1 === '1'); // false (different types)
4. What is hoisting in JavaScript? ›

Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their scope before code execution.

Example:

console.log(x); // undefined (due to hoisting)
var x = 5;
5. What is an IIFE in JavaScript? ›

An IIFE (Immediately Invoked Function Expression) is a function that is executed immediately after it is defined.

Example:

(function() {
    console.log("IIFE executed!");
})();
6. What is the difference between null and undefined? ›

null: A value explicitly assigned to a variable to represent "no value".

undefined: A variable that has been declared but not assigned any value.

Example:

let a;
console.log(a); // undefined

let b = null;
console.log(b); // null
7. What is a callback function in JavaScript? ›

A callback function is a function passed into another function as an argument and executed after the parent function completes its task.

Example:

function greet(name, callback) {
    console.log('Hello ' + name);
    callback();
}

function sayGoodbye() {
    console.log('Goodbye!');
}

greet('John', sayGoodbye);
8. What are promises in JavaScript? ›

A promise is an object that represents the eventual completion or failure of an asynchronous operation.

Example:

let promise = new Promise(function(resolve, reject) {
    let success = true;
    if(success) {
        resolve('Promise fulfilled!');
    } else {
        reject('Promise rejected!');
    }
});

promise.then(function(message) {
    console.log(message);
}).catch(function(error) {
    console.log(error);
});
9. What is the 'this' keyword in JavaScript? ›

The 'this' keyword refers to the context in which a function is called. It can represent different objects depending on how the function is invoked.

Example:

const obj = {
    name: 'Alice',
    greet: function() {
        console.log('Hello, ' + this.name);
    }
};

obj.greet(); // Hello, Alice
10. What is event delegation? ›

Event delegation is a technique where a single event listener is added to a parent element instead of multiple listeners on child elements. It utilizes event bubbling to handle events.

Example:

const parent = document.getElementById('parent');
parent.addEventListener('click', function(event) {
    if (event.target.tagName === 'BUTTON') {
        console.log('Button clicked: ' + event.target.innerText);
    }
});
11. What is the purpose of the 'bind' method? ›

The 'bind' method creates a new function that, when called, has its 'this' keyword set to the provided value, with a given sequence of arguments preceding any provided when the new function is called.

Example:

const obj = {
    name: 'Alice'
};

function greet() {
    console.log('Hello, ' + this.name);
}

const greetAlice = greet.bind(obj);
greetAlice(); // Hello, Alice
12. Explain the concept of prototype in JavaScript. ›

In JavaScript, every object has a prototype, which is another object from which it can inherit properties and methods. This allows for shared behavior between instances of objects.

Example:

function Person(name) {
    this.name = name;
}

Person.prototype.greet = function() {
    console.log('Hello, ' + this.name);
};

const alice = new Person('Alice');
alice.greet(); // Hello, Alice
13. What are the different data types in JavaScript? ›

JavaScript has the following data types:

  • Primitive Types: String, Number, Boolean, Null, Undefined, Symbol, BigInt
  • Reference Types: Object, Array, Function
14. What is the difference between an object and an array? ›

An object is a collection of key-value pairs, while an array is an ordered collection of values. Arrays can be thought of as a special type of object.

Example:

const obj = { name: 'Alice', age: 25 }; // Object
const arr = [1, 2, 3, 4, 5]; // Array
15. What is a higher-order function in JavaScript? ›

A higher-order function is a function that takes another function as an argument, returns a function, or both.

Example:

function map(arr, callback) {
    const result = [];
    for (let i = 0; i < arr.length; i++) {
        result.push(callback(arr[i]));
    }
    return result;
}

const numbers = [1, 2, 3];
const squares = map(numbers, x => x * x);
console.log(squares); // [1, 4, 9]
16. What is the 'new' keyword in JavaScript? ›

The 'new' keyword is used to create an instance of an object that has a constructor function. It allocates memory and binds 'this' to the new object.

Example:

function Car(make, model) {
    this.make = make;
    this.model = model;
}

const myCar = new Car('Toyota', 'Corolla');
console.log(myCar); // Car { make: 'Toyota', model: 'Corolla' }
17. What is an arrow function in JavaScript? ›

An arrow function is a shorter syntax for writing function expressions in JavaScript. It does not have its own 'this', which means it lexically binds 'this' from the surrounding code.

Example:

const add = (a, b) => a + b;
console.log(add(2, 3)); // 5
18. What is the spread operator in JavaScript? ›

The spread operator (...) allows an iterable (like an array) to be expanded in places where 0+ arguments or elements are expected. It can be used for function calls, array literals, and object literals.

Example:

const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];
const combined = [...arr1, ...arr2];
console.log(combined); // [1, 2, 3, 4, 5, 6]
19. What is destructuring in JavaScript? ›

Destructuring is a JavaScript expression that makes it possible to unpack values from arrays or properties from objects into distinct variables.

Example:

const arr = [1, 2, 3];
const [a, b] = arr;
console.log(a); // 1
console.log(b); // 2

const obj = { x: 1, y: 2 };
const { x, y } = obj;
console.log(x); // 1
console.log(y); // 2
20. What is the purpose of the 'typeof' operator? ›

The 'typeof' operator is used to determine the type of a variable or expression. It returns a string indicating the type of the unevaluated operand.

Example:

console.log(typeof 'Hello'); // "string"
console.log(typeof 42); // "number"
console.log(typeof true); // "boolean"
console.log(typeof undefined); // "undefined"
console.log(typeof null); // "object" (this is a known bug)
21. What is the difference between synchronous and asynchronous programming? ›

Synchronous programming is when tasks are executed one after another, blocking the execution of the next task until the current one is complete. Asynchronous programming allows tasks to be executed concurrently, enabling the program to continue running while waiting for operations to complete.

22. What is AJAX? ›

AJAX (Asynchronous JavaScript and XML) is a technique for creating asynchronous web applications. It allows web pages to be updated asynchronously by exchanging data with a web server behind the scenes.

23. What are template literals in JavaScript? ›

Template literals are a way to work with strings in JavaScript, allowing for multi-line strings and string interpolation using backticks (``).

Example:

const name = 'Alice';
const greeting = `Hello, ${name}!`;
console.log(greeting); // Hello, Alice!
24. What is a Promise chain? ›

A Promise chain is a sequence of Promise calls where each call returns a new Promise, allowing you to perform asynchronous operations in a clean and manageable way.

Example:

fetch('https://api.example.com/data')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.log('Error:', error));
25. What are modules in JavaScript? ›

Modules are a way to encapsulate code in JavaScript, allowing you to divide your code into reusable pieces. Each module can export its functionality and import others, promoting better organization and maintenance.

Example:

// module.js
export const PI = 3.14;
export function add(x, y) {
    return x + y;
}

// main.js
import { PI, add } from './module.js';
console.log(PI); // 3.14
console.log(add(2, 3)); // 5
26. What is the difference between 'call', 'apply', and 'bind'? ›

'call', 'apply', and 'bind' are methods that control the 'this' context of a function:

  • call: Calls a function with a given 'this' value and arguments provided individually.
  • apply: Calls a function with a given 'this' value and arguments provided as an array.
  • bind: Returns a new function with a given 'this' value, allowing you to set 'this' for future calls.
Example:

function greet(greeting) {
    console.log(greeting + ', ' + this.name);
}

const obj = { name: 'Alice' };

greet.call(obj, 'Hello'); // Hello, Alice
greet.apply(obj, ['Hi']); // Hi, Alice

const greetAlice = greet.bind(obj);
greetAlice('Hey'); // Hey, Alice
27. What is the 'this' keyword in JavaScript? ›

The 'this' keyword refers to the context in which a function is called. Its value can change based on how the function is invoked (e.g., as a method, a standalone function, or an event handler).

28. Explain the concept of 'event bubbling' in JavaScript. ›

Event bubbling is a mechanism in JavaScript where an event starts from the target element and bubbles up to the parent elements in the DOM. This allows parent elements to handle events triggered by their child elements.

29. What is the difference between forEach and map? ›

forEach: Executes a provided function once for each array element but does not return a new array.

map: Creates a new array populated with the results of calling a provided function on every element in the calling array.

Example:

const numbers = [1, 2, 3];

const squares = numbers.map(num => num * num);
console.log(squares); // [1, 4, 9]

numbers.forEach(num => console.log(num)); // 1, 2, 3
30. What is a closure in JavaScript? ›

A closure is a function that retains access to its lexical scope, even when the function is executed outside that scope. This allows for data encapsulation and private variables.

Example:

function makeCounter() {
    let count = 0;
    return function() {
        count++;
        return count;
    };
}

const counter = makeCounter();
console.log(counter()); // 1
console.log(counter()); // 2
31. What is a JavaScript function expression? ›

A function expression is a function that is defined within an expression, usually assigned to a variable. Unlike function declarations, function expressions can be anonymous.

Example:

const greet = function(name) {
    return 'Hello, ' + name;
};

console.log(greet('Alice')); // Hello, Alice
32. What is the difference between a function declaration and a function expression? ›

A function declaration defines a function with a name and is hoisted, allowing it to be called before its definition. A function expression creates a function without a name (or with one) and is not hoisted.

Example:

// Function Declaration
function greet() {
    return 'Hello';
}

// Function Expression
const sayHello = function() {
    return 'Hello';
};

greet(); // Hello
sayHello(); // Hello
33. Explain the concept of 'hoisting' in JavaScript. ›

Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their containing scope during the compile phase, allowing them to be used before their declaration.

Example:

console.log(x); // undefined
var x = 5;

function sayHi() {
    console.log('Hi');
}
sayHi(); // Hi
34. What is an Immediately Invoked Function Expression (IIFE)? ›

An IIFE is a function that is defined and executed immediately after its creation. It helps in avoiding polluting the global scope and is commonly used for encapsulation.

Example:

(function() {
    console.log('I am an IIFE');
})(); // I am an IIFE
35. What are async and await in JavaScript? ›

Async and await are used for handling asynchronous operations in a cleaner way. An async function returns a Promise, and await can be used to pause the execution until the Promise is resolved.

Example:

async function fetchData() {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
}

fetchData();
36. What is the difference between '==' and '===' in JavaScript? ›

'==' is the equality operator that checks for value equality with type coercion, while '===' is the strict equality operator that checks for both value and type equality without type coercion.

Example:

console.log(5 == '5'); // true (type coercion)
console.log(5 === '5'); // false (strict equality)
37. What is JSON? ›

JSON (JavaScript Object Notation) is a lightweight data interchange format that is easy to read and write for humans and easy to parse and generate for machines. It is often used for transmitting data in web applications.

38. How can you parse a JSON string in JavaScript? ›

You can parse a JSON string using the JSON.parse() method, which converts a JSON string into a JavaScript object.

Example:

const jsonString = '{"name": "Alice", "age": 25}';
const obj = JSON.parse(jsonString);
console.log(obj.name); // Alice
39. What is the purpose of the JSON.stringify() method? ›

The JSON.stringify() method converts a JavaScript object into a JSON string, which can then be sent to a server or stored in a file.

Example:

const obj = { name: 'Alice', age: 25 };
const jsonString = JSON.stringify(obj);
console.log(jsonString); // {"name":"Alice","age":25}
40. What is the 'window' object in JavaScript? ›

The 'window' object represents the browser's window and is the global object in a browser environment. It provides properties and methods for interacting with the browser, such as alert(), console.log(), and setTimeout().
