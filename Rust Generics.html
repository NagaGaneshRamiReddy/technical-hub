Rust Generics

Understanding Generics in Rust

Generics in Rust allow you to write flexible and reusable code. By using generics, you can define functions, structs, enums, and traits with placeholder types that can be specified later, enhancing code abstraction and reusability.

Generic Functions

Generic functions allow you to create functions that work with different data types. Here’s an example of a generic function that finds the largest item in a slice:

fn largest(items: &[T]) -> &T {
    let mut largest = &items[0];

    for item in items {
        if item > largest {
            largest = item;
        }
    }

    largest
}

fn main() {
    let numbers = vec![34, 50, 25, 100, 65];
    let result = largest(&numbers);
    println!("The largest number is {}", result);
}

Generic Structs

You can define structs with generic types, allowing them to hold data of any type. For example:

struct Point {
    x: T,
    y: T,
}

fn main() {
    let integer_point = Point { x: 5, y: 10 };
    let float_point = Point { x: 1.0, y: 4.0 };

    println!("Integer Point: ({}, {})", integer_point.x, integer_point.y);
    println!("Float Point: ({}, {})", float_point.x, float_point.y);
}

Generic Enums

Enums can also use generics. This allows for powerful data modeling. Here’s an example:

enum Option {
    Some(T),
    None,
}

fn main() {
    let some_number = Option::Some(5);
    let no_number: Option = Option::None;
}

Trait Bounds

When using generics, you can specify constraints using trait bounds. This ensures that the generic type implements specific behaviors:

fn print_length(item: T) {
    println!("Length: {}", item);
}

fn main() {
    print_length("Hello, Rust!");
}

Using the Where Clause

For more complex scenarios, you can use the where clause to specify bounds more clearly:

fn compare(a: T, b: U) 
    where T: PartialOrd + std::fmt::Debug, U: std::fmt::Debug {
        if a > b {
            println!("{:?} is greater", a);
        } else {
            println!("{:?} is greater", b);
        }
    }

fn main() {
    compare(10, 5);
}

Summary

Generics in Rust enable code reusability and flexibility by allowing functions and types to operate on multiple data types. By using generics, you can create more abstract and general solutions while maintaining type safety.
