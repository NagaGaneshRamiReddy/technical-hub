Dynamic memory allocation in C

Introduction

Dynamic memory allocation in C allows you to allocate and deallocate memory at runtime. This capability is crucial when dealing with data structures whose size is not known until the program is running. C provides several standard functions for dynamic memory management: malloc, calloc, realloc, and free.

malloc (Memory Allocation)

The malloc function allocates a specified number of bytes of memory and returns a pointer to the beginning of the allocated memory block. If the allocation fails, it returns NULL.

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;

    // Allocate memory for an integer
    ptr = (int *)malloc(sizeof(int));

    if (ptr == NULL) {
        printf("Memory allocation failed.\n");
        return 1;
    }

    *ptr = 10; // Use the allocated memory
    printf("Value: %d\n", *ptr);

    free(ptr); // Free the allocated memory
    return 0;
}

calloc (Contiguous Allocation)

The calloc function allocates memory for an array of elements and initializes all bits to zero. It takes two arguments: the number of elements and the size of each element. Like malloc, if the allocation fails, it returns NULL.

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *arr;
    int n = 5;

    // Allocate memory for an array of 5 integers
    arr = (int *)calloc(n, sizeof(int));

    if (arr == NULL) {
        printf("Memory allocation failed.\n");
        return 1;
    }

    for (int i = 0; i < n; i++) {
        printf("Element %d: %d\n", i, arr[i]);
    }

    free(arr); // Free the allocated memory
    return 0;
}

realloc (Reallocation)

The realloc function resizes a previously allocated memory block. It is used to extend or reduce the size of the memory block. It returns a pointer to the newly allocated memory, which may be at a different location. If the reallocation fails, it returns NULL and the original memory block remains unchanged.

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *arr;
    int n = 5;

    // Allocate memory for an array of 5 integers
    arr = (int *)malloc(n * sizeof(int));

    if (arr == NULL) {
        printf("Memory allocation failed.\n");
        return 1;
    }

    // Initialize array
    for (int i = 0; i < n; i++) {
        arr[i] = i;
    }

    // Reallocate memory to hold 10 integers
    arr = (int *)realloc(arr, 10 * sizeof(int));

    if (arr == NULL) {
        printf("Memory reallocation failed.\n");
        return 1;
    }

    // Initialize new elements
    for (int i = 5; i < 10; i++) {
        arr[i] = i;
    }

    for (int i = 0; i < 10; i++) {
        printf("Element %d: %d\n", i, arr[i]);
    }

    free(arr); // Free the allocated memory
    return 0;
}

free (Deallocation)

The free function deallocates a previously allocated memory block. It is crucial to call free to release memory that is no longer needed to prevent memory leaks. Passing a NULL pointer to free is safe and has no effect.

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;

    // Allocate memory for an integer
    ptr = (int *)malloc(sizeof(int));

    if (ptr == NULL) {
        printf("Memory allocation failed.\n");
        return 1;
    }

    *ptr = 10; // Use the allocated memory

    // Free the allocated memory
    free(ptr);

    return 0;
}

Best Practices

  • Always check if memory allocation was successful by verifying if the returned pointer is NULL.
  • Use free to release memory when it is no longer needed.
  • Be cautious of memory leaks and dangling pointers by ensuring all allocated memory is freed.

Summary

Dynamic memory allocation in C provides the flexibility to manage memory during runtime. By using malloc, calloc, realloc, and free, you can efficiently handle memory for various data structures and avoid memory leaks.
