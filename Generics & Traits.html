Generics & Traits

Understanding Generics in Rust

Generics allow you to write flexible, reusable functions and data structures. They enable you to define a function or a type with placeholder types, making your code more abstract and adaptable.

Using Generics in Functions

Hereâ€™s an example of a generic function that finds the largest element in a slice:

fn largest(items: &[T]) -> &T {
            let mut largest = &items[0];
        
            for item in items {
                if item > largest {
                    largest = item;
                }
            }
        
            largest
        }
        
        fn main() {
            let numbers = vec![34, 50, 25, 100, 65];
            let result = largest(&numbers);
            println!("The largest number is {}", result);
        }

Using Generics in Structs

Generics can also be used in structs to create data types that can operate on multiple types:

struct Point {
            x: T,
            y: T,
        }
        
        fn main() {
            let integer_point = Point { x: 5, y: 10 };
            let float_point = Point { x: 1.0, y: 4.0 };
        
            println!("Integer Point: ({}, {})", integer_point.x, integer_point.y);
            println!("Float Point: ({}, {})", float_point.x, float_point.y);
        }

Understanding Traits

Traits are a way to define shared behavior in Rust. A trait defines functionality that can be shared across types, similar to interfaces in other languages.

Defining and Implementing Traits

Hereâ€™s how to define a trait and implement it for a struct:

trait Summary {
            fn summarize(&self) -> String;
        }
        
        struct Article {
            headline: String,
            content: String,
        }
        
        impl Summary for Article {
            fn summarize(&self) -> String {
                format!("{}: {}", self.headline, self.content)
            }
        }
        
        fn main() {
            let article = Article {
                headline: String::from("Generics in Rust"),
                content: String::from("An introduction to generics and traits."),
            };
        
            println!("Article Summary: {}", article.summarize());
        }

Traits as Bounds

Traits can also be used as bounds on generic types. This allows you to restrict the types that can be used with a generic function:

fn notify(item: T) {
            println!("Breaking news! {}", item.summarize());
        }
        
        fn main() {
            let article = Article {
                headline: String::from("Generics in Rust"),
                content: String::from("An introduction to generics and traits."),
            };
        
            notify(article);
        }

Summary

Generics and traits are powerful features in Rust that enable code reusability and abstraction. By using generics, you can create flexible functions and data types, while traits allow you to define shared behavior across different types.
