Passing Data Between Screens

Passing Data in Flutter

When navigating between screens in Flutter, you can pass data from one screen to another, enabling dynamic content display based on user interactions.

1. Basic Data Passing

Data can be passed directly when pushing a new screen using the constructor of the target screen.

Navigator.push(
  context,
  MaterialPageRoute(
    builder: (context) => SecondScreen(data: 'Hello from FirstScreen!'),
  ),
);

Expected Output:

The SecondScreen will display the passed data ("Hello from FirstScreen!") on the screen.

2. Implementing the Second Screen

Here’s how you can implement the SecondScreen to receive and display the passed data:

class SecondScreen extends StatelessWidget {
  final String data;

  SecondScreen({required this.data});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Second Screen')),
      body: Center(child: Text(data)),
    );
  }
}

3. Practical Example

Consider a scenario where a user selects an item from a list, and you want to show more details about that item on the next screen.

ListTile(
  title: Text('Item 1'),
  onTap: () {
    Navigator.push(
      context,
      MaterialPageRoute(
        builder: (context) => DetailScreen(data: 'Details about Item 1'),
      ),
    );
  },
);

Expected Output:

Tapping on "Item 1" navigates to DetailScreen displaying "Details about Item 1".

Summary

Passing data between screens in Flutter enhances the interactivity of applications. You can use constructors to send data and display it effectively on the next screen, making your app more dynamic.
