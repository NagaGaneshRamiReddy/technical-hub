Controlling Visibility with pub

Understanding Visibility in Rust

In Rust, controlling visibility is crucial for managing access to your code. The pub keyword is used to make items (functions, structs, enums, etc.) accessible from other modules. By default, items are private to the module they are defined in.

Using the pub Keyword

To make an item public, you prefix it with pub. Here’s a simple example:

mod my_module {
    pub fn public_function() {
        println!("This function is public!");
    }

    fn private_function() {
        println!("This function is private!");
    }
}

fn main() {
    my_module::public_function(); // This works
    // my_module::private_function(); // This will cause a compile error
}

Visibility for Structs and Enums

Structs and enums can also control visibility for their fields. Here’s how you can do it:

mod my_module {
    pub struct PublicStruct {
        pub field1: i32,  // Public field
        field2: i32,      // Private field
    }
    
    pub enum PublicEnum {
        Variant1,
        Variant2,
    }
}

fn main() {
    let my_struct = my_module::PublicStruct { field1: 10, field2: 20 };
    println!("Field1: {}", my_struct.field1);
    // println!("Field2: {}", my_struct.field2); // This will cause a compile error
}

Nested Modules and Visibility

When working with nested modules, you can control visibility at multiple levels. Here’s an example:

mod outer {
    pub mod inner {
        pub fn inner_function() {
            println!("This is an inner function!");
        }
    }
}

fn main() {
    outer::inner::inner_function(); // This works
    // outer::inner::private_function(); // This will cause a compile error
}

Re-exporting Items

You can re-export items to control visibility further and simplify access using pub use:

mod my_module {
    pub mod inner {
        pub fn inner_function() {
            println!("Inner function!");
        }
    }

    pub use inner::inner_function; // Re-exporting the function
}

fn main() {
    my_module::inner_function(); // Now accessible directly
}

Summary

Controlling visibility in Rust is essential for managing access to your code and maintaining encapsulation. The pub keyword allows you to specify which items are accessible from outside their defining modules, ensuring that your code remains organized and secure.
