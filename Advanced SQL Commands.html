Advanced SQL Commands

Advanced SQL Commands

Advanced SQL commands provide powerful ways to manage and manipulate data beyond the basic CRUD operations. These commands include complex querying techniques, data manipulation features, and optimization tools to enhance the efficiency and flexibility of SQL operations.

Common Advanced SQL Commands

1. JOIN Operations

JOIN operations are used to combine rows from two or more tables based on a related column. Common types of JOINs include:

  • INNER JOIN - Retrieves rows with matching values in both tables.
  • LEFT JOIN (or LEFT OUTER JOIN) - Retrieves all rows from the left table and matched rows from the right table; returns NULL for unmatched rows from the right table.
  • RIGHT JOIN (or RIGHT OUTER JOIN) - Retrieves all rows from the right table and matched rows from the left table; returns NULL for unmatched rows from the left table.
  • FULL JOIN (or FULL OUTER JOIN) - Retrieves all rows when there is a match in either table; returns NULL for unmatched rows in both tables.
    SELECT a.*, b.*
    FROM table1 a
    INNER JOIN table2 b ON a.common_column = b.common_column;

2. UNION and UNION ALL

The UNION and UNION ALL commands are used to combine the results of two or more SELECT queries:

  • UNION - Combines results and removes duplicate rows.
  • UNION ALL - Combines results and includes all rows, including duplicates.
    SELECT column1 FROM table1
    UNION
    SELECT column1 FROM table2;

    SELECT column1 FROM table1
    UNION ALL
    SELECT column1 FROM table2;

3. Subqueries

A subquery is a query nested inside another query. Subqueries can be used in SELECT, INSERT, UPDATE, and DELETE statements:

  • Scalar Subquery - Returns a single value.
  • Column Subquery - Returns a single column of values.
  • Row Subquery - Returns a single row of values.
  • Table Subquery - Returns a result set with multiple rows and columns.
    SELECT emp_name
    FROM employees
    WHERE salary > (SELECT AVG(salary) FROM employees);

4. Transactions

Transactions are used to manage a group of SQL statements as a single unit. Transactions ensure that all operations are completed successfully or none are applied. Key commands include:

  • BEGIN TRANSACTION - Starts a new transaction.
  • COMMIT - Saves all changes made during the transaction.
  • ROLLBACK - Reverts all changes made during the transaction.
    BEGIN TRANSACTION;
    UPDATE employees SET salary = salary * 1.1 WHERE department = 'IT';
    INSERT INTO audit_log (event) VALUES ('Salary updated for IT department');
    COMMIT;

5. Stored Procedures

Stored procedures are precompiled collections of SQL statements that can be executed as a single unit. They are used to encapsulate complex operations and improve performance:

    CREATE PROCEDURE IncreaseSalary
    @dept NVARCHAR(50),
    @percentage FLOAT
    AS
    BEGIN
        UPDATE employees
        SET salary = salary * (1 + @percentage)
        WHERE department = @dept;
    END;

    EXEC IncreaseSalary 'Sales', 0.05;

6. Triggers

Triggers are special procedures that are automatically executed in response to certain events on a table, such as INSERT, UPDATE, or DELETE:

    CREATE TRIGGER trgAfterInsert
    ON employees
    FOR INSERT
    AS
    BEGIN
        INSERT INTO audit_log (event, details)
        SELECT 'New employee added', emp_name FROM inserted;
    END;

Summary

Advanced SQL commands enable more sophisticated data manipulation and management, from combining data across multiple tables with JOINs to automating tasks with triggers and stored procedures. Mastering these commands can significantly enhance your ability to work with complex datasets and optimize database performance.
